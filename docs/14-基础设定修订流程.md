## 一、流程概述

`reviseFoundation` 用于在书籍已有基础设定后，根据用户反馈重新生成设定。它将旧设定和用户反馈注入 ArchitectAgent，生成符合当前 5 段 SECTION 格式的新设定，同时保留原有世界观/角色/主线/伏笔/语气，不改动已写章节的运行时事实。

```
reviseFoundation(bookId, feedback)
  │
  ├─ 1. 判断布局版本（Phase5 / Phase4）
  │
  ├─ 2. 备份现有基础设定
  │   ├─ Phase5: 备份 outline/ + roles/（递归）
  │   └─ Phase4: 备份 4 个扁平文件
  │
  ├─ 3. 读取现有设定
  │   ├─ Phase5: 读 outline/ + roles/ + book_rules.md
  │   └─ Phase4: 读 4 个扁平文件
  │
  ├─ 4. ArchitectAgent.generateFoundation(book, reviseFrom={旧设定, userFeedback})
  │   ├─ 将旧设定重组为当前 5 段 SECTION 格式
  │   ├─ 保留原有世界观/角色/主线/伏笔/语气
  │   └─ 不改动已写章节的运行时事实
  │
  ├─ 5. Foundation Reviewer 审查
  │   ├─ 审查失败 → 仅警告，不阻断
  │   └─ 审查异常 → 仅警告，跳过
  │
  └─ 6. writeFoundationFiles(mode="revise")
      ├─ 清空旧 roles/ 目录
      └─ 重新写入 outline/ + roles/ + book_rules.md
```

## 二、Phase5 与 Phase4 布局差异

inkos 经历了布局演进，`reviseFoundation` 需要同时兼容两种布局：

### Phase4（旧布局）- 扁平文件

```
story/
  story_bible.md         ← 世界观设定
  volume_outline.md      ← 卷大纲
  book_rules.md          ← 书籍规则
  character_matrix.md    ← 角色矩阵
```

### Phase5（新布局）- 目录结构

```
story/
  outline/
    story_frame.md       ← 世界观铁律（替代 story_bible.md）
    volume_map.md        ← 卷地图（替代 volume_outline.md）
  roles/
    主要角色/             ← 主要角色目录
      {角色名}.md
    次要角色/             ← 次要角色目录
      {角色名}.md
  book_rules.md          ← 书籍规则（保留）
```

**判断方式**：`isNewLayoutBook(bookDir)` 检测是否存在 `outline/` 目录。

## 三、reviseFoundation 完整流程

### 3.1 备份现有基础设定

在修改前，将现有设定备份到带时间戳的目录：

```
story/.backup-{phase}-{timestamp}/
```

其中 `{phase}` 为 `phase5` 或 `phase4`，`{timestamp}` 为 ISO 时间戳（`:` 和 `.` 替换为 `-`）。

#### Phase5 备份

```go
backupDir := filepath.Join(storyDir, fmt.Sprintf(".backup-phase5-%s", timestamp))
os.MkdirAll(backupDir, 0755)

// 备份扁平文件（兼容部分迁移的书）
for _, fileName := range []string{"story_bible.md", "volume_outline.md", "book_rules.md", "character_matrix.md"} {
    src := filepath.Join(storyDir, fileName)
    if content, err := os.ReadFile(src); err == nil {
        os.WriteFile(filepath.Join(backupDir, fileName), content, 0644)
    }
    // 缺失的 legacy shim 文件可以安全跳过
}

// 备份 outline/ 目录（浅拷贝）
copyDirShallow(filepath.Join(storyDir, "outline"), filepath.Join(backupDir, "outline"))

// 备份 roles/ 目录（递归拷贝）
copyDirRecursive(filepath.Join(storyDir, "roles"), filepath.Join(backupDir, "roles"))
```

#### Phase4 备份

```go
backupDir := filepath.Join(storyDir, fmt.Sprintf(".backup-phase4-%s", timestamp))
os.MkdirAll(backupDir, 0755)

// 仅备份 4 个扁平文件
for _, fileName := range []string{"story_bible.md", "volume_outline.md", "book_rules.md", "character_matrix.md"} {
    src := filepath.Join(storyDir, fileName)
    if content, err := os.ReadFile(src); err == nil {
        os.WriteFile(filepath.Join(backupDir, fileName), content, 0644)
    }
}
```

### 3.2 读取现有设定

根据布局版本读取不同的文件来源：

#### Phase5 读取

```go
// Phase5+: 读 outline/ + roles/ + book_rules.md（权威来源）
oldStoryBible, _ := readStoryFrame(bookDir)        // outline/story_frame.md
oldVolumeOutline, _ := readVolumeMap(bookDir)       // outline/volume_map.md
oldCharacterMatrix, _ := readCharacterContext(bookDir) // roles/ 目录聚合
oldBookRules, _ := os.ReadFile(filepath.Join(storyDir, "book_rules.md"))
```

#### Phase4 读取

```go
// Legacy: 读 4 个扁平文件
oldStoryBible, _ := os.ReadFile(filepath.Join(storyDir, "story_bible.md"))
oldVolumeOutline, _ := os.ReadFile(filepath.Join(storyDir, "volume_outline.md"))
oldBookRules, _ := os.ReadFile(filepath.Join(storyDir, "book_rules.md"))
oldCharacterMatrix, _ := os.ReadFile(filepath.Join(storyDir, "character_matrix.md"))
```

**关键差异**：Phase5 优先读取 `outline/` 和 `roles/` 中的权威文件，而非兼容 shim 扁平文件，否则大段角色/故事细节可能在重写中丢失。

### 3.3 ArchitectAgent.generateFoundation(reviseFrom)

将旧设定和用户反馈注入 ArchitectAgent，生成新设定：

```go
architect := NewArchitectAgent(agentCtx)
foundation, err := architect.GenerateFoundation(book, nil, nil, &GenerateFoundationOptions{
    ReviseFrom: &ReviseFrom{
        StoryBible:      oldStoryBible,
        VolumeOutline:   oldVolumeOutline,
        BookRules:       oldBookRules,
        CharacterMatrix: oldCharacterMatrix,
        UserFeedback:    feedback,
    },
})
```

#### ReviseFrom 结构

```go
type ReviseFrom struct {
    StoryBible      string // 旧世界观设定
    VolumeOutline   string // 旧卷大纲
    BookRules       string // 旧书籍规则
    CharacterMatrix string // 旧角色矩阵
    UserFeedback    string // 用户修订反馈
}
```

#### 生成约束

ArchitectAgent 在 `reviseFrom` 模式下遵循以下约束：

1. **格式重组**：将旧设定重组为当前 5 段 SECTION 格式（详见第四节）
2. **保留原有内容**：
   - 保留原有世界观铁律
   - 保留原有角色设定
   - 保留原有主线情节
   - 保留原有伏笔布局
   - 保留原有语气风格
3. **不改动运行时事实**：
   - 不改动已写章节中确立的事实
   - 不改动 `current_state.md` 中的当前状态
   - 不改动 `pending_hooks.md` 中的伏笔状态
4. **应用用户反馈**：根据 `userFeedback` 调整设定方向

### 3.4 Foundation Reviewer 审查

生成新设定后，调用 FoundationReviewerAgent 审查：

```go
reviewer := NewFoundationReviewerAgent(agentCtx)
resolvedLanguage := LangZH
if book.Language == "en" {
    resolvedLanguage = LangEN
}

review, err := reviewer.Review(FoundationReviewParams{
    Foundation:       foundation,
    Mode:             "original",
    Language:         resolvedLanguage,
    TargetChapters:   book.TargetChapters,
})

if err != nil {
    // 审查异常 → 仅警告，不阻断
    logger.Warn("[reviseFoundation] Foundation review failed and was skipped: %s", err)
} else if !review.Passed {
    // 审查失败 → 仅警告，不阻断（接受重写结果）
    logger.Warn("[reviseFoundation] Foundation review did not pass; accepting rewrite. Feedback: %s", review.OverallFeedback)
}
```

**关键语义**：修订流程中 Foundation Reviewer 的审查结果**仅作为警告**，不会阻断设定写入。这与建书流程中审查失败会阻断不同——因为修订是在已有设定基础上的调整，即使审查不完美也优于旧设定。

### 3.5 writeFoundationFiles(mode="revise")

审查通过（或跳过）后，将新设定写入磁盘：

```go
outlineDir := filepath.Join(storyDir, "outline")
os.MkdirAll(outlineDir, 0755)
os.MkdirAll(filepath.Join(storyDir, "roles", "主要角色"), 0755)
os.MkdirAll(filepath.Join(storyDir, "roles", "次要角色"), 0755)

genreProfile, _ := loadGenreProfile(book.Genre)
architect.WriteFoundationFiles(bookDir, foundation, genreProfile.NumericalSystem, book.Language, "revise")
```

#### mode="revise" 的特殊行为

`writeFoundationFiles` 的 `mode` 参数为 `"revise"` 时，执行以下特殊逻辑：

1. **清空旧 roles/ 目录**：删除 `roles/主要角色/` 和 `roles/次要角色/` 下的所有文件，然后重新写入
2. **覆盖 outline/ 文件**：覆盖 `outline/story_frame.md` 和 `outline/volume_map.md`
3. **覆盖 book_rules.md**：覆盖 `story/book_rules.md`

```
writeFoundationFiles(mode="revise")
  │
  ├─ 清空 roles/主要角色/ 目录
  ├─ 清空 roles/次要角色/ 目录
  │
  ├─ 写入 outline/story_frame.md ← foundation.StoryFrame
  ├─ 写入 outline/volume_map.md ← foundation.VolumeMap
  ├─ 写入 book_rules.md ← foundation.BookRules
  │
  ├─ for each character in foundation.Characters:
  │   ├─ 主要角色 → roles/主要角色/{name}.md
  │   └─ 次要角色 → roles/次要角色/{name}.md
  │
  └─ 写入 character_matrix.md（兼容 shim）
```

## 四、SECTION 5 段格式

ArchitectAgent 生成的基础设定使用统一的 5 段 SECTION 格式。`reviseFrom` 模式会将旧设定重组为以下格式：

### SECTION 1: 世界观铁律（Story Frame）

```markdown
# 世界观铁律

## 世界规则
{世界运行的基础规则}

## 力量体系
{力量/等级/修炼体系}

## 地理设定
{主要地点和地理关系}

## 势力格局
{主要势力和关系}

## 时间线
{历史大事件和时间节点}
```

### SECTION 2: 角色设定（Characters）

```markdown
# 角色设定

## 主要角色

### {角色名}
- **身份**: {身份}
- **性格**: {性格特征}
- **能力**: {能力}
- **动机**: {核心动机}
- **关系**: {与其他角色的关系}
- **弧光**: {角色成长弧线}

## 次要角色

### {角色名}
- **身份**: {身份}
- **作用**: {在故事中的作用}
```

### SECTION 3: 主线情节（Volume Map）

```markdown
# 卷大纲

## 第一卷: {卷名}
- **核心冲突**: {本卷核心冲突}
- **主要事件**: {事件列表}
- **卷末高潮**: {高潮事件}
- **伏笔播种**: {本卷播种的伏笔}
- **伏笔回收**: {本卷回收的伏笔}

## 第二卷: {卷名}
...
```

### SECTION 4: 伏笔布局（Hooks）

```markdown
# 伏笔布局

## 核心伏笔
| ID | 类型 | 播种章节 | 预期回收 | 状态 |
|----|------|----------|----------|------|
| H01 | 身世之谜 | 第1章 | 第三卷 | open |

## 次要伏笔
| ID | 类型 | 播种章节 | 预期回收 | 状态 |
|----|------|----------|----------|------|
| H02 | 物品线索 | 第3章 | 第一卷末 | open |
```

### SECTION 5: 语气风格（Book Rules）

```markdown
# 书籍规则

## 叙事语气
{语气描述}

## 节奏要求
{节奏规则}

## 禁忌事项
{不能出现的内容}

## 章节结构
{章节结构规范}

## 数值系统
{数值/等级系统规则}
```

## 五、修订约束

### 不改动的文件

`reviseFoundation` **不改动**以下运行时文件：

| 文件 | 说明 |
|------|------|
| `story/current_state.md` | 当前状态（由已写章节确立） |
| `story/pending_hooks.md` | 伏笔池（由已写章节确立） |
| `story/particle_ledger.md` | 粒子台账（由已写章节确立） |
| `story/chapter_summaries.md` | 章节摘要（由已写章节确立） |
| `story/state/*.json` | 结构化运行时状态 |
| `story/snapshots/` | 状态快照 |
| `chapters/` | 已写章节正文 |
| `story/author_intent.md` | 作者意图 |
| `story/current_focus.md` | 当前聚焦 |
| `story/style_guide.md` | 风格指南 |

### 改动的文件

| 文件 | 操作 |
|------|------|
| `story/outline/story_frame.md` | 覆盖 |
| `story/outline/volume_map.md` | 覆盖 |
| `story/book_rules.md` | 覆盖 |
| `story/roles/主要角色/*.md` | 清空后重写 |
| `story/roles/次要角色/*.md` | 清空后重写 |
| `story/character_matrix.md` | 覆盖（兼容 shim） |
| `story/story_bible.md` | 覆盖（兼容 shim，Phase4） |
| `story/volume_outline.md` | 覆盖（兼容 shim，Phase4） |

### 备份目录

| 目录 | 说明 |
|------|------|
| `story/.backup-phase5-{timestamp}/` | Phase5 备份（outline/ + roles/ + 扁平文件） |
| `story/.backup-phase4-{timestamp}/` | Phase4 备份（4 个扁平文件） |

// Foundation 基础设定产出
type Foundation struct {
	StoryFrame      string         // 世界观铁律
	Characters      []CharacterSpec // 角色列表
	VolumeMap       string         // 卷大纲
	BookRules       string         // 书籍规则
	CharacterMatrix string         // 角色矩阵（兼容 shim）
	Hooks           []HookSpec     // 伏笔布局
}

// CharacterSpec 角色规格
type CharacterSpec struct {
	Name        string // 角色名
	Role        string // 主要角色 / 次要角色
	Identity    string // 身份
	Personality string // 性格
	Ability     string // 能力
	Motivation  string // 动机
	Relations   string // 关系
	Arc         string // 弧光
	Content     string // 完整角色 Markdown
}

// HookSpec 伏笔规格
type HookSpec struct {
	HookID         string // 伏笔 ID
	Type           string // 类型
	StartChapter   int    // 播种章节
	ExpectedPayoff string // 预期回收
	PayoffTiming   string // 回收时机
	Notes          string // 备注
	CoreHook       bool   // 是否核心伏笔
}

// ReviseFrom 修订来源
type ReviseFrom struct {
	StoryBible      string `json:"storyBible"`
	VolumeOutline   string `json:"volumeOutline"`
	BookRules       string `json:"bookRules"`
	CharacterMatrix string `json:"characterMatrix"`
	UserFeedback    string `json:"userFeedback"`
}

// GenerateFoundationOptions 生成选项
type GenerateFoundationOptions struct {
	ReviseFrom *ReviseFrom `json:"reviseFrom,omitempty"`
}

		language string,
		mode string, // "original" 或 "revise"
	) error
}

## 附：修订流程时序图

```
reviseFoundation(bookId, feedback)
  │
  ├─ 1. isNewLayoutBook(bookDir) → isPhase5
  │
  ├─ 2. BackupFoundation(bookDir, isPhase5)
  │     ├─ Phase5: backup outline/ + roles/ + flatFiles
  │     └─ Phase4: backup 4 flatFiles
  │     → backupDir = story/.backup-{phase}-{timestamp}/
  │
  ├─ 3. ReadExistingFoundation(bookDir, isPhase5)
  │     ├─ Phase5: readStoryFrame + readVolumeMap + readCharacterContext + readBookRules
  │     └─ Phase4: read 4 flatFiles
  │     → {oldStoryBible, oldVolumeOutline, oldBookRules, oldCharacterMatrix}
  │
  ├─ 4. state.loadBookConfig(bookId) → book
  │
  ├─ 5. architect.GenerateFoundation(book, reviseFrom={
  │       storyBible: oldStoryBible,
  │       volumeOutline: oldVolumeOutline,
  │       bookRules: oldBookRules,
  │       characterMatrix: oldCharacterMatrix,
  │       userFeedback: feedback
  │     })
  │     → foundation
  │
  ├─ 6. reviewer.Review({foundation, mode:"original", language, targetChapters})
  │     ├─ review.passed=false → logger.Warn（仅警告，不阻断）
  │     └─ error → logger.Warn（仅警告，跳过审查）
  │
  ├─ 7. mkdir outline/ + roles/主要角色/ + roles/次要角色/
  │
  └─ 8. architect.WriteFoundationFiles(bookDir, foundation, numericalSystem, language, "revise")
        ├─ 清空 roles/ 目录
        ├─ 写入 outline/story_frame.md
        ├─ 写入 outline/volume_map.md
        ├─ 写入 book_rules.md
        ├─ 写入 roles/主要角色/{name}.md
        ├─ 写入 roles/次要角色/{name}.md
        └─ 写入 character_matrix.md（兼容 shim）
```
