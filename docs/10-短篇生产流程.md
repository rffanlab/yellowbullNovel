## 一、流程概述

短篇生产是独立于 PipelineRunner 的流水线，不复用长篇的章节写作循环。短篇一次性写完全部章节（12-18 章，每章 900-1200 字），经过一轮审查+修订后产出最终稿，再生成卖点包装和封面。

```
direction（创作方向）
  │
  ▼
┌─────────────────────────────────────────────────────┐
│ 大纲阶段（3 轮）                                     │
│                                                      │
│  ShortFictionOutlineAgent.createOutline              │
│    → outline v001（rawContent + storyTitle）         │
│  ShortFictionOutlineReviewerAgent.reviewOutline      │
│    → review markdown                                 │
│  ShortFictionOutlineReviserAgent.reviseOutline       │
│    → outline v002（采纳审查意见修订）                 │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ 写作阶段                                             │
│                                                      │
│  ShortFictionWriterAgent.writeDraft                  │
│    → draft v001（一次性写完全部章节）                 │
│  if 有缺失章节:                                       │
│    continueDraft（最多3次补写）                      │
│  validateShortFictionDraftForFinal                   │
│    → 校验章节数 + 空章节                             │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ 审查阶段                                             │
│                                                      │
│  ShortFictionDraftReviewerAgent.reviewDraft          │
│    → draft review markdown                           │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ 修订阶段                                             │
│                                                      │
│  ShortFictionDraftReviserAgent.reviseDraft           │
│    → draft v002（采纳审查意见修订）                   │
│  if 修订失败/不完整:                                  │
│    保留 v001，记录 warning                           │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ 最终产物                                             │
│                                                      │
│  writeFinalArtifacts                                 │
│    → final/full.md（完整正文）                       │
│    → final/short-story.json（结构化数据）             │
│    → final/chapters/0001.md ... 0018.md              │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ 卖点包装                                             │
│                                                      │
│  ShortFictionPackagingAgent.generatePackage          │
│    → title / intro / sellingPoints / coverPrompt     │
│  writePackageArtifacts                               │
│    → final/sales-package.md                          │
│    → final/sales-package.json                        │
│    → final/cover-prompt.md                           │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ 封面生成（可选）                                     │
│                                                      │
│  generateCoverArtifact                               │
│    → final/cover.png（或 cover.jpg）                 │
│  if 失败: coverError = error message                 │
└────────────────────────┬────────────────────────────┘
                         ▼
              ShortFictionRunResult
```

**核心设计原则**：
1. **独立流水线**：短篇不复用 PipelineRunner，没有 truth files / governed 上下文 / 章节索引
2. **一次性写作**：全部章节在一次 LLM 调用中写完，缺失章节用 continueDraft 补写
3. **单轮修订**：只做一轮审查+一轮修订，不像长篇有循环审查
4. **失败保留 v001**：修订失败时不覆盖完整首稿，保留 v001 作为最终稿

## 二、入口函数签名

```go
// RunShortFictionProduction 执行短篇生产流水线。
//
// 参数：
//   - options.ProjectRoot：项目根目录
//   - options.Direction：创作方向（一句话描述）
//   - options.Runtimes：各阶段的 AgentContext（planner/outlineReview/writer/draftReview/revise/package）
//   - options.Reference：参考资料（可选）
//   - options.StoryID：故事 ID（用于断点续跑，nil 则自动生成）
//   - options.OutDir：输出目录（默认 "shorts"）
//   - options.ChapterCount：章节数（12-18，默认 12）
//   - options.CharsPerChapter：每章字数（zh: 900-1200, en: 600-800）
//   - options.Language：语言（"zh" / "en"）
//   - options.Cover：是否生成封面（默认 false）
//   - options.OnProgress：进度回调
//
// 返回：
//   - *ShortFictionRunResult：产物路径列表
//   - error
func RunShortFictionProduction(ctx context.Context, options *ShortFictionRunOptions) (*ShortFictionRunResult, error)
```

## 三、详细步骤

### 3.1 大纲阶段（3 轮）

大纲阶段分 3 步：生成 → 审查 → 修订。

**Step 1：生成大纲 v001**

```go
outlineAgent := NewShortFictionOutlineAgent(options.Runtimes.Planner)
outlineV1, err := outlineAgent.CreateOutline(ctx, &ShortFictionOutlineInput{
    Direction:       options.Direction,
    ChapterCount:    chapterCount,
    CharsPerChapter: charsPerChapter,
    Reference:       options.Reference,
    Language:        language,
})
// outlineV1 = { StoryTitle, RawContent }

// 如果没有提供 storyId，从标题生成
if providedStoryID == "" {
    storyID = safeSegment(slugify(outlineV1.StoryTitle))
} else {
    storyID = providedStoryID
}
baseDir := filepath.Join(outDir, storyID)

// 保存 outline v001
writeText(projectRoot, filepath.Join(baseDir, "outline", "v001.md"), outlineV1.RawContent)
```

**LLM 参数**：`temperature=0.55, maxTokens=8192`

**Step 2：审查大纲**

```go
outlineReviewer := NewShortFictionOutlineReviewerAgent(options.Runtimes.OutlineReview)
outlineReview, err := outlineReviewer.ReviewOutline(ctx, &ShortFictionOutlineReviewInput{
    Direction: options.Direction,
    Outline:   outlineV1,
    Reference: options.Reference,
    Language:  language,
})
// outlineReview = review markdown 文本

// 保存审查报告
writeText(projectRoot, filepath.Join(baseDir, "reviews", "outline-v001.md"), outlineReview)
```

**LLM 参数**：`temperature=0.3, maxTokens=4096`

**Step 3：修订大纲 v002**

```go
outlineReviser := NewShortFictionOutlineReviserAgent(options.Runtimes.Planner)
outlineV2, err := outlineReviser.ReviseOutline(ctx, &ShortFictionOutlineRevisionInput{
    Direction:       options.Direction,
    Outline:         outlineV1,
    Review:          outlineReview,
    Reference:       options.Reference,
    ChapterCount:    chapterCount,
    CharsPerChapter: charsPerChapter,
    Language:        language,
})
// outlineV2 = { StoryTitle, RawContent }（采纳审查意见后的修订版）

// 保存 outline v002
writeText(projectRoot, filepath.Join(baseDir, "outline", "v002.md"), outlineV2.RawContent)
outlineMarkdown := outlineV2.RawContent
```

**LLM 参数**：`temperature=0.45, maxTokens=8192`

**修订的多轮对话结构**：
```
system: buildShortFictionOutlineSystemPrompt
user:   buildShortFictionOutlineUserPrompt（原始方向+约束）
assistant: outlineV1.RawContent（首版大纲）
user:   buildShortFictionOutlineRevisionFollowup（审查意见+修订要求）
→ LLM 输出 outlineV2
```

### 3.2 写作阶段

**一次性写完全部章节**：

```go
writer := NewShortFictionWriterAgent(options.Runtimes.Writer)
draftV1, err := writer.WriteDraft(ctx, &ShortFictionDraftInput{
    Direction:       options.Direction,
    OutlineMarkdown: outlineMarkdown,
    ChapterCount:    chapterCount,
    CharsPerChapter: charsPerChapter,
    Language:        language,
})
// draftV1 = { StoryTitle, OpeningHook, Chapters[], RawContent }
```

**LLM 参数**：`temperature=0.58, maxTokens=estimateShortFictionMaxTokens(chapterCount, charsPerChapter)`

**maxTokens 估算**：
```go
func estimateShortFictionMaxTokens(chapterCount, charsPerChapter int) int {
    // 总字数 × 1.5（含格式开销和 markdown 结构）
    return chapterCount * charsPerChapter * 3 / 2
}
```

**补写缺失章节**：

LLM 可能漏写部分章节，用 `continueDraft` 补写：

```go
const ShortFictionDraftCompletionAttempts = 3

missingFromDraft := findEmptyShortFictionChapters(draftV1)
if len(missingFromDraft) > 0 {
    // 保存 partial 版本
    writeDraftArtifacts(projectRoot, baseDir, "v001-partial", draftV1, language)

    for attempt := 1; len(missingFromDraft) > 0 && attempt <= ShortFictionDraftCompletionAttempts; attempt++ {
        draftV1, err = writer.ContinueDraft(ctx, &ShortFictionDraftContinuationInput{
            Direction:       options.Direction,
            OutlineMarkdown: outlineMarkdown,
            ChapterCount:    chapterCount,
            CharsPerChapter: charsPerChapter,
            Language:        language,
            Draft:           draftV1, // 传入已有草稿
        })
        missingFromDraft = findEmptyShortFictionChapters(draftV1)
        if len(missingFromDraft) > 0 {
            writeDraftArtifacts(projectRoot, baseDir, "v001-partial", draftV1, language)
        }
    }
}
```

**continueDraft 的多轮对话结构**：
```
system: buildShortFictionWriterSystemPrompt
user:   buildShortFictionDraftContinuationUserPrompt
        （含 existingDraftMarkdown + missingChapters 列表）
→ LLM 输出缺失章节，拼接到 draftV1.RawContent
```

**LLM 参数**：`temperature=0.68`（比首次写作高，增加多样性），`maxTokens=estimateShortFictionMaxTokens(missingCount, charsPerChapter)`

**校验 + 保存 v001**：

```go
// 校验：章节数必须匹配 + 不能有空章节
validateShortFictionDraftForFinal(draftV1, chapterCount)
// 如果校验失败，抛出错误（导致整个流程失败，status.json 标记 failed）

// 保存 v001
writeDraftArtifacts(projectRoot, baseDir, "v001", draftV1, language)
```

**draft 产物目录结构**：
```
shorts/{storyId}/
  drafts/
    v001/
      full.md          # 完整正文（markdown）
      draft.json       # 结构化数据
      chapters/
        0001.md        # 第1章
        0002.md        # 第2章
        ...
        0018.md        # 第18章
    v001-partial/      # 补写过程中的中间产物（如果有）
      ...
```

### 3.3 审查阶段

```go
draftReviewer := NewShortFictionDraftReviewerAgent(options.Runtimes.DraftReview)
draftReview, err := draftReviewer.ReviewDraft(ctx, &ShortFictionDraftReviewInput{
    Direction:       options.Direction,
    OutlineMarkdown: outlineMarkdown,
    Draft:           draftV1,
    ChapterCount:    chapterCount,
    CharsPerChapter: charsPerChapter,
    Language:        language,
})
// draftReview = review markdown 文本

// 保存审查报告
writeText(projectRoot, filepath.Join(baseDir, "reviews", "draft-v001.md"), draftReview)
```

**LLM 参数**：`temperature=0.3, maxTokens=8192`

**审查维度**（由 prompt 定义）：
- 开篇钩子吸引力
- 节奏控制（12-18章的短篇要求紧凑）
- 情绪曲线
- 反转设计
- 角色一致性
- 主题表达
- 结尾力度

### 3.4 修订阶段

```go
reviser := NewShortFictionDraftReviserAgent(options.Runtimes.Revise)
var finalDraft *ShortFictionBatchDraft
var revisionWarning string

finalDraft = draftV1 // 默认用 v001

draftV2, err := reviser.ReviseDraft(ctx, &ShortFictionDraftRevisionInput{
    Direction:       options.Direction,
    OutlineMarkdown: outlineMarkdown,
    Draft:           draftV1,
    Review:          draftReview,
    ChapterCount:    chapterCount,
    CharsPerChapter: charsPerChapter,
    Language:        language,
})

if err != nil {
    // 修订失败：保留 v001，记录 warning
    revisionWarning = err.Error()
    writeText(projectRoot, filepath.Join(baseDir, "reviews", "draft-v002-warning.md"),
        "# 第二轮改稿未采用\n\n系统没有用不完整或解析失败的改稿覆盖完整首稿。\n\n## 原因\n\n"+revisionWarning)
} else {
    // 校验 v2 完整性
    if err := validateShortFictionDraftForFinal(draftV2, chapterCount); err != nil {
        revisionWarning = err.Error()
        writeText(...)
    } else {
        // v2 完整，采用
        writeDraftArtifacts(projectRoot, baseDir, "v002", draftV2, language)
        finalDraft = draftV2
    }
}
```

**LLM 参数**：`temperature=0.45, maxTokens=estimateShortFictionMaxTokens(chapterCount, charsPerChapter)`

**修订的多轮对话结构**：
```
system: buildShortFictionWriterSystemPrompt
user:   buildShortFictionWriterUserPrompt（原始方向+约束）
assistant: draftV1.RawContent（首版草稿）
user:   buildShortFictionDraftRevisionFollowup（审查意见+修订要求）
→ LLM 输出 draftV2
```

**失败保留 v001 的设计**：
- 修订可能产生不完整或解析失败的输出
- 用不完整的改稿覆盖完整首稿是灾难性的
- 因此：v2 校验失败 → 保留 v001 作为 finalDraft，记录 warning

### 3.5 最终产物

```go
// 生成最终产物
writeFinalArtifacts(projectRoot, baseDir, finalDraft, language)
```

**final 目录结构**：
```
shorts/{storyId}/
  final/
    full.md                    # 完整正文（markdown，含标题+开篇钩子+全部章节）
    {safeFileName}.md          # 以标题命名的副本
    short-story.json           # 结构化数据（= ShortFictionBatchDraft）
    chapters/
      0001.md                  # 第1章（# 标题 + 正文）
      0002.md
      ...
      0018.md
```

**writeFinalArtifacts 实现**：
```go
func writeFinalArtifacts(root, baseDir string, draft *ShortFictionBatchDraft, language string) error {
    finalDir := filepath.Join(baseDir, "final")

    // 1. full.md（完整 markdown）
    markdown := renderShortFictionDraftMarkdown(draft, language)
    writeText(root, filepath.Join(finalDir, "full.md"), markdown)

    // 2. 以标题命名的副本
    writeText(root, filepath.Join(finalDir, safeFileName(draft.StoryTitle)+".md"), markdown)

    // 3. short-story.json（结构化数据）
    writeJSON(root, filepath.Join(finalDir, "short-story.json"), draft)

    // 4. 各章节独立文件
    for _, chapter := range draft.Chapters {
        content := fmt.Sprintf("# %s\n\n%s",
            formatShortFictionChapterHeading(chapter.Number, chapter.Title, language),
            chapter.Content)
        writeText(root, filepath.Join(finalDir, "chapters",
            fmt.Sprintf("%04d.md", chapter.Number)), content)
    }
    return nil
}
```

简介：{简介}

## 四、storyId 断点续跑逻辑

`storyId` 是短篇的唯一标识，用于断点续跑。提供相同 storyId 重跑时，会跳过已完成的阶段。

### 4.1 续跑决策

```go
func RunShortFictionProduction(ctx context.Context, options *ShortFictionRunOptions) (*ShortFictionRunResult, error) {
    root := options.ProjectRoot
    outDir := normalizeOutputDir(options.OutDir) // 默认 "shorts"
    providedStoryID := ""
    if options.StoryID != "" {
        providedStoryID = safeSegment(options.StoryID)
    }

    // 检查是否已完成（final/full.md 存在且 status.json 不是 failed）
    if providedStoryID != "" {
        finalPath := filepath.Join(outDir, providedStoryID, "final", "full.md")
        if projectFileExists(root, finalPath) && !isFailedShortRun(root, filepath.Join(outDir, providedStoryID, "status.json")) {
            // 已完成，直接返回结果
            return buildShortRunResult(providedStoryID, filepath.Join(outDir, providedStoryID),
                CoverError: "already-complete"), nil
        }
    }

    // 执行生产流程
    result, err := produceShort(options, root, outDir, providedStoryID)
    if err != nil {
        // 标记为 failed
        if providedStoryID != "" {
            writeJSON(root, filepath.Join(outDir, providedStoryID, "status.json"), map[string]interface{}{
                "status": "failed",
                "error":  err.Error(),
            })
        }
        return nil, err
    }
    return result, nil
}
```

### 4.2 大纲续跑

```go
func produceShort(options, root, outDir, providedStoryID) (*ShortFictionRunResult, error) {
    // ... 参数校验 ...

    // 尝试读取已有的 outline v002
    var resumedOutline string
    if providedStoryID != "" {
        resumedOutline, _ = tryReadProjectText(root,
            filepath.Join(outDir, providedStoryID, "outline", "v002.md"))
    }

    var outlineMarkdown string
    var storyID, baseDir string

    if providedStoryID != "" && strings.TrimSpace(resumedOutline) != "" {
        // 大纲已存在，跳过大纲3轮
        storyID = providedStoryID
        baseDir = filepath.Join(outDir, storyID)
        outlineMarkdown = resumedOutline
        options.OnProgress("Resuming from existing outline (skipping outline stages)...")
    } else {
        // 执行完整大纲3轮
        // ... createOutline → reviewOutline → reviseOutline ...
    }

    // ... 写作/审查/修订/包装 ...
}
```

### 4.3 续跑场景

| 场景 | 检测条件 | 续跑行为 |
|------|----------|----------|
| 已完成 | `final/full.md` 存在 + status 非 failed | 直接返回结果 |
| 大纲完成 | `outline/v002.md` 存在 | 跳过大纲3轮，从写作开始 |
| 写作中断 | 无 v002，有 v001-partial | 重新执行大纲+写作（partial 不复用） |
| 失败 | status.json = failed | 重新执行全部流程 |

### 4.4 status.json

```json
{
  "status": "complete",  // "complete" / "failed"
  "warning": "revision skipped: ...",  // 可选
  "updatedAt": "2025-07-11T12:00:00.000Z"
}
```

失败时：
```json
{
  "status": "failed",
  "error": "Short-hit draft is incomplete; expected 12 chapters, got 8.",
  "updatedAt": "2025-07-11T12:00:00.000Z"
}
```

## 五、短篇与长篇的区别

| 对比项 | 短篇 | 长篇 |
|--------|------|------|
| **流水线** | 独立 `runShortFictionProduction` | `PipelineRunner` |
| **章节数** | 12-18 章 | 不限（目标章数由 book.targetChapters 定） |
| **每章字数** | zh: 900-1200 字, en: 600-800 词 | zh: 3000+ 字, en: 2000+ 词 |
| **总字数** | ~12000-21600 字 | 10万-数百万字 |
| **写作方式** | 一次性写完全部章节 | 逐章写作（writeNextChapter） |
| **大纲** | 3 轮（生成→审查→修订） | ArchitectAgent 生成（可选审查循环） |
| **审查** | 1 轮 | 循环审查（最多 maxRetries 轮） |
| **修订** | 1 轮（失败保留 v001） | 循环修订（auto/polish/rewrite/...） |
| **truth files** | 无 | 10+ 个（state/ledger/hooks/summaries/...） |
| **governed 上下文** | 无 | 有（chapterIntent/contextPackage/ruleStack） |
| **章节索引** | 无 | 有（ChapterMeta[]） |
| **卷级合并** | 无 | 有（ConsolidatorAgent） |
| **断点续跑** | storyId + status.json | resumeFrom + snapshotState |
| **回炉重写** | 无 | 有（6种模式） |
| **AI 检测** | 无 | 有（detectAndRewrite） |
| **产物** | full.md + short-story.json + sales-package + cover | chapters/000N.md + truth files |
| **封面** | 可选（generateCoverArtifact） | 无（长篇封面由其他流程处理） |

**短篇不需要 truth files 的原因**：
1. 短篇一次性写完，不需要跨章上下文累积
2. 全部章节在一次 LLM 调用中生成，上下文就是大纲+方向
3. 没有"续写"场景（长篇需要 truth files 是因为逐章写作需要前序章节的状态）

**短篇一次性写作的上下文管理**：
- 输入：direction + outlineMarkdown + chapterCount + charsPerChapter
- 输出：全部章节的完整正文
- maxTokens 估算：`chapterCount × charsPerChapter × 1.5`
- 对于 18 章 × 1200 字 = 21600 字，maxTokens ≈ 32400，在 LLM 窗口内

## 附录：完整流程时序

```
RunShortFictionProduction(options)
  │
  ├─ 检查 storyId 是否已完成
  │   └─ if final/full.md 存在 && status != failed: return 已完成结果
  │
  ├─ produceShort(options)
  │   │
  │   ├─ 参数校验
  │   │   ├─ chapterCount: 12-18（默认12）
  │   │   └─ charsPerChapter: zh 900-1200 / en 600-800
  │   │
  │   ├─ 检查 outline v002 是否存在（断点续跑）
  │   │   ├─ 存在: 跳过大纲3轮，outlineMarkdown = resumedOutline
  │   │   └─ 不存在: 执行大纲3轮
  │   │       ├─ CreateOutline → v001（temp=0.55）
  │   │       ├─ ReviewOutline → review（temp=0.3）
  │   │       └─ ReviseOutline → v002（temp=0.45）
  │   │
  │   ├─ 写作阶段
  │   │   ├─ WriteDraft → v001（temp=0.58，一次性写完）
  │   │   ├─ if 有缺失章节:
  │   │   │   └─ ContinueDraft 循环（最多3次，temp=0.68）
  │   │   ├─ ValidateShortFictionDraftForFinal（校验完整性）
  │   │   └─ WriteDraftArtifacts("v001")
  │   │
  │   ├─ 审查阶段
  │   │   ├─ ReviewDraft → review（temp=0.3）
  │   │   └─ 保存 reviews/draft-v001.md
  │   │
  │   ├─ 修订阶段
  │   │   ├─ finalDraft = v001（默认）
  │   │   ├─ ReviseDraft → v002（temp=0.45）
  │   │   ├─ if 成功 && 校验通过:
  │   │   │   ├─ WriteDraftArtifacts("v002")
  │   │   │   └─ finalDraft = v002
  │   │   └─ else: 保留 v001，记录 warning
  │   │
  │   ├─ WriteFinalArtifacts(finalDraft)
  │   │   └─ final/full.md + short-story.json + chapters/
  │   │
  │   ├─ 卖点包装
  │   │   ├─ GeneratePackage → salesPackage（temp=0.45）
  │   │   └─ WritePackageArtifacts
  │   │
  │   └─ 封面生成（可选）
  │       ├─ if options.Cover == false: coverError = "disabled"
  │       └─ else: generateCoverArtifact
  │           ├─ resolveCoverGenerationRequest（解析 API 配置）
  │           ├─ buildCoverImagePrompt（构建封面 prompt）
  │           ├─ 调用图片生成 API（gemini/images/responses）
  │           └─ 保存 final/cover.png
  │
  ├─ if revisionWarning: writeShortRunStatus("complete", warning)
  │
  └─ return buildShortRunResult(storyId, baseDir, coverArtifacts)
```
