## 一、流程概述

卷级合并是长篇小说的上下文压缩机制。当一本书写到几百章时，`chapter_summaries.md` 会变得极其庞大，导致每次写作时加载的上下文超出 LLM 窗口。ConsolidatorAgent 在每卷写完后，将该卷的逐章摘要压缩成一段 ≤500 字的叙事段落，归档原始详细摘要，只保留当前卷的逐章行。

```
chapter_summaries.md（逐章摘要表）
  │
  ▼
┌─────────────────────────────────────────────────────┐
│ ConsolidatorAgent.Consolidate(bookDir)              │
│                                                      │
│  1. 读取 chapter_summaries.md + volume_map.md       │
│  2. parseVolumeBoundaries → 卷边界列表              │
│  3. rerunAdvancedCountPromotion → 伏笔晋升检查      │
│  4. 识别已完成卷（endCh <= maxChapter）              │
│  5. 对每个已完成卷：                                  │
│     LLM 压缩章节摘要 → ≤500字叙事段落               │
│  6. 归档旧摘要到 summaries_archive/                  │
│  7. 重写 chapter_summaries.md（仅保留当前卷行）      │
│  8. 追加写入 volume_summaries.md                    │
└────────────────────────┬────────────────────────────┘
                         ▼
              ConsolidationResult
  { volumeSummaries, archivedVolumes,
    retainedChapters, promotedHookCount }
```

**核心设计原则**：
1. **只压缩已完成卷**：当前卷的逐章摘要必须保留，因为写作时需要精确的上一章状态
2. **归档不删除**：原始摘要归档到 `summaries_archive/` 目录，可用于回溯
3. **伏笔晋升独立执行**：即使没有已完成卷，`rerunAdvancedCountPromotion` 仍然执行
4. **幂等性**：重复调用不会重复压缩同一卷（已完成卷的行已被归档清空）

### 3.1 读取 chapter_summaries.md + volume_map.md

```go
func (a *ConsolidatorAgent) Consolidate(ctx context.Context, bookDir string) (*ConsolidationResult, error) {
    storyDir := filepath.Join(bookDir, "story")
    summariesPath := filepath.Join(storyDir, "chapter_summaries.md")
    volumeSummariesPath := filepath.Join(storyDir, "volume_summaries.md")

    // 并行读取两个文件
    summariesRaw, _ := os.ReadFile(summariesPath)    // 不存在则返回空
    outlineRaw, _ := readVolumeMap(bookDir)          // 读取 volume_map.md（fallback: volume_outline.md）

    // Phase 7 hotfix 2: 伏笔晋升检查（独立于摘要合并，总是执行）
    promotedHookCount := a.rerunAdvancedCountPromotion(storyDir)

    // 如果任一文件缺失，直接返回（伏笔晋升已执行）
    if len(summariesRaw) == 0 || len(outlineRaw) == 0 {
        return &ConsolidationResult{
            VolumeSummaries:    "",
            ArchivedVolumes:    0,
            RetainedChapters:   0,

        }, nil
    }
```

### 3.2 parseVolumeBoundaries 解析卷边界

从 volume_map.md 中解析每卷的名称和章节范围。

```go
// VolumeBoundary 卷边界
type VolumeBoundary struct {
    Name    string // 卷名，如 "第一卷：山野少年"
    StartCh int    // 起始章节号
    EndCh   int    // 结束章节号
}

// parseVolumeBoundaries 从 outline 文本中解析卷边界。
//
// 匹配规则：
//   1. 行首匹配卷标题：第N卷 / Volume N
//   2. 行中匹配章节范围：(第1-30章) / (Chapters 1-30) / 第1-30章
func (a *ConsolidatorAgent) parseVolumeBoundaries(outline string) []VolumeBoundary {
    var volumes []VolumeBoundary

    // 卷标题正则：第X卷 / Volume N
    volumeHeader := regexp.MustCompile(`(?i)^(第[一二三四五六七八九十百千万零〇\d]+卷|Volume\s+\d+)`)
    // 章节范围正则：(第1-30章) / (Chapters 1-30) / 第1-30章
    rangePattern := regexp.MustCompile(`(?i)[（(]\s*(?:第|[Cc]hapters?\s+)?(\d+)\s*[-–~～—]\s*(\d+)\s*(?:章)?\s*[）)]|(?:第|[Cc]hapters?\s+)(\d+)\s*[-–~～—]\s*(\d+)\s*(?:章)?`)

    for _, rawLine := range strings.Split(outline, "\n") {
        // 去掉 markdown 标题前缀
        line := strings.TrimPrefix(strings.TrimSpace(rawLine), "#")
        line = strings.TrimSpace(line)
        if !volumeHeader.MatchString(line) {
            continue
        }

        rangeMatch := rangePattern.FindStringSubmatch(line)
        if rangeMatch == nil {
            continue
        }

        // rangeMatch[1]/[2] 对应括号内范围，[3]/[4] 对应无括号范围
        startCh, _ := strconv.Atoi(firstNonEmpty(rangeMatch[1], rangeMatch[3]))
        endCh, _ := strconv.Atoi(firstNonEmpty(rangeMatch[2], rangeMatch[4]))
        if startCh <= 0 || endCh <= 0 {
            continue
        }

        // 卷名 = 范围标记前的文本
        rangeIndex := rangePattern.FindStringIndex(line)[0]
        name := strings.TrimSpace(strings.TrimRight(line[:rangeIndex], "（( "))
        if name != "" {
            volumes = append(volumes, VolumeBoundary{Name: name, StartCh: startCh, EndCh: endCh})
        }
    }
    return volumes
}
```

**解析示例**：

输入：
```markdown
## 第一卷：山野少年（第1-30章）
## 第二卷：青云风云（Chapters 31-60）
## 第三卷：筑基风云 (第 61-90 章)
```

输出：
```go
[]VolumeBoundary{
    {Name: "第一卷：山野少年", StartCh: 1,  EndCh: 30},
    {Name: "第二卷：青云风云", StartCh: 31, EndCh: 60},
    {Name: "第三卷：筑基风云", StartCh: 61, EndCh: 90},
}
```

如果解析结果为空（没有卷边界），直接返回，不做合并：

```go
volumeBoundaries := a.parseVolumeBoundaries(outlineRaw)

        VolumeSummaries:   "",
        ArchivedVolumes:   0,
        RetainedChapters:  0,
        PromotedHookCount: promotedHookCount,
    }, nil
}
```

### 3.3 识别已完成卷

```go
// parseSummaryTable 解析章节摘要表
func (a *ConsolidatorAgent) parseSummaryTable(raw string) (header string, rows []SummaryRow) {
    lines := strings.Split(raw, "\n")

    // 表头行：包含 "章节"/"Chapter" 或 "---"
    var headerLines []string
    var dataLines []string
    for _, line := range lines {
        if !strings.HasPrefix(line, "|") {
            continue
        }
        if strings.Contains(line, "章节") || strings.Contains(line, "Chapter") || strings.Contains(line, "---") {
            headerLines = append(headerLines, line)
        } else {
            dataLines = append(dataLines, line)
        }
    }
    header = strings.Join(headerLines, "\n")

    // 数据行：提取章节号
    chapterPattern := regexp.MustCompile(`\|\s*(\d+)\s*\|`)
    for _, line := range dataLines {
        match := chapterPattern.FindStringSubmatch(line)
        if match != nil {
            chNum, _ := strconv.Atoi(match[1])
            if chNum > 0 {
                rows = append(rows, SummaryRow{Chapter: chNum, Raw: line})
            }
        }
    }
    return header, rows
}
```

**识别已完成卷**：

```go
header, rows := a.parseSummaryTable(string(summariesRaw))
if len(rows) == 0 {
    return &ConsolidationResult{...}, nil
}

// 计算最大章节号
maxChapter := 0
for _, r := range rows {
    if r.Chapter > maxChapter {
        maxChapter = r.Chapter
    }
}

// 分类：已完成卷 vs 当前卷
type CompletedVolume struct {
    Name    string
    StartCh int
    EndCh   int
    Rows    []SummaryRow
}

var completedVolumes []CompletedVolume
var currentVolumeRows []SummaryRow

for _, vol := range volumeBoundaries {
    var volRows []SummaryRow
    for _, r := range rows {
        if r.Chapter >= vol.StartCh && r.Chapter <= vol.EndCh {
            volRows = append(volRows, r)
        }
    }

    // 已完成卷：endCh <= maxChapter 且有摘要行
    if vol.EndCh <= maxChapter && len(volRows) > 0 {
        completedVolumes = append(completedVolumes, CompletedVolume{
            Name: vol.Name, StartCh: vol.StartCh, EndCh: vol.EndCh, Rows: volRows,
        })
    } else {
        // 当前卷/未完成卷：保留详细行
        currentVolumeRows = append(currentVolumeRows, volRows...)
    }
}

// 保留不被任何卷覆盖的行（卷边界可能不全）
coveredChapters := make(map[int]bool)
for _, vol := range volumeBoundaries {
    for ch := vol.StartCh; ch <= vol.EndCh; ch++ {
        coveredChapters[ch] = true
    }
}
for _, r := range rows {
    if !coveredChapters[r.Chapter] {
        currentVolumeRows = append(currentVolumeRows, r)
    }
}

// 没有已完成卷则返回
if len(completedVolumes) == 0 {
    return &ConsolidationResult{
        VolumeSummaries:   "",
        ArchivedVolumes:   0,
        RetainedChapters:  len(currentVolumeRows),
        PromotedHookCount: promotedHookCount,
    }, nil
}
```

### 3.4 rerunAdvancedCountPromotion 伏笔晋升检查

**Phase 7 hotfix 2**：在归档前重新检查伏笔晋升。即使没有已完成卷，这一步也会执行。

```go
// rerunAdvancedCountPromotion 重新运行 advanced_count 伏笔晋升检查。
//
// 伏笔晋升规则：当一个伏笔（hook）的 advancedCount（已推进章节数）
// 超过阈值（默认2章）时，将其 promoted 标志从 false 翻转为 true。
//
// 这一步独立于摘要合并，因为：
//   1. 新书可能没有已完成卷，但伏笔仍需晋升
//   2. 归档后摘要表会变短，可能影响 advancedCount 计算
//
// 返回：本次翻转的伏笔数量
func (a *ConsolidatorAgent) rerunAdvancedCountPromotion(storyDir string) (int, error) {
    ledgerPath := filepath.Join(storyDir, "pending_hooks.md")
    raw, err := os.ReadFile(ledgerPath)
    if err != nil || len(strings.TrimSpace(string(raw))) == 0 {
        return 0, nil // 文件不存在或为空
    }

    // 解析伏笔表
    hooks := parsePendingHooksMarkdown(string(raw))
    if len(hooks) == 0 {
        return 0, nil
    }

    // 检测语言
    language := detectLanguage(string(raw)) // "zh" / "en"

    // 读取章节摘要（用于计算 advancedCount）
    summariesRaw, _ := os.ReadFile(filepath.Join(storyDir, "chapter_summaries.md"))

    // 执行晋升检查
    result := rerunPromotionPass(hooks, string(summariesRaw))
    if !result.Updated {
        return 0, nil
    }

    // 写回伏笔表
    rendered := renderHookSnapshot(result.Hooks, language)
    os.WriteFile(ledgerPath, []byte(rendered), 0644)

    return result.FlippedCount, nil
}
```

**rerunPromotionPass 逻辑**：

```go
// rerunPromotionPass 检查所有伏笔的 advancedCount 是否超过阈值。
//
// advancedCount 计算：扫描 chapter_summaries.md，统计每个伏笔
// 在摘要中被提及/推进的章节数。
//
// 晋升条件：advancedCount >= 2（默认阈值）
type PromotionResult struct {
    Hooks        []StoredHook // 更新后的伏笔列表
    Updated      bool         // 是否有变化
    FlippedCount int          // 本次翻转的数量
}

func rerunPromotionPass(hooks []StoredHook, summariesRaw string) PromotionResult {
    flipped := 0
    updated := false

    for i := range hooks {
        if hooks[i].Promoted {
            continue // 已经晋升过
        }

        // 计算 advancedCount
        advancedCount := countHookAdvancements(hooks[i], summariesRaw)

        // 超过阈值则晋升
        if advancedCount >= 2 {
            hooks[i].Promoted = true
            hooks[i].AdvancedCount = advancedCount
            flipped++
            updated = true
        }
    }

    return PromotionResult{
        Hooks:        hooks,
        Updated:      updated,
        FlippedCount: flipped,
    }
}
```

### 3.5 LLM 压缩章节摘要为叙事段落

对每个已完成卷，用 LLM 将逐章摘要压缩成一段 ≤500 字的叙事段落。

```go
// 读取已有的 volume_summaries.md（追加模式）
existingVolSummaries, _ := os.ReadFile(volumeSummariesPath)
var newSummaries []string
if len(existingVolSummaries) > 0 {
    newSummaries = append(newSummaries, strings.TrimSpace(string(existingVolSummaries)))
} else {
    newSummaries = append(newSummaries, "# Volume Summaries\n") // 首次创建
}

// 对每个已完成卷调用 LLM
for _, vol := range completedVolumes {
    // 拼接该卷的所有摘要行
    volSummaryRows := make([]string, len(vol.Rows))
    for i, r := range vol.Rows {
        volSummaryRows[i] = r.Raw
    }

    // LLM 压缩
    systemPrompt := "You are a narrative summarizer. Compress chapter-by-chapter " +
        "summaries into a single coherent paragraph (max 500 words) that captures " +
        "the key events, character developments, and plot progression of this volume. " +
        "Preserve specific names, locations, and plot points. " +
        "Write in the same language as the input."

    userPrompt := fmt.Sprintf("Volume: %s (Chapters %d-%d)\n\nChapter summaries:\n%s\n%s",
        vol.Name, vol.StartCh, vol.EndCh, header, strings.Join(volSummaryRows, "\n"))

    response, err := a.chat(ctx, []Message{
        {Role: "system", Content: systemPrompt},
        {Role: "user",   Content: userPrompt},
    }, ChatOptions{Temperature: 0.3})
    if err != nil {
        return nil, err
    }

    // 追加到摘要列表
    newSummaries = append(newSummaries, fmt.Sprintf("\n## %s (Ch.%d-%d)\n\n%s",
        vol.Name, vol.StartCh, vol.EndCh, strings.TrimSpace(response.Content)))
}
```

**LLM 参数**：`temperature=0.3`（低温度保证忠实原文）

**输出格式**：

```markdown
# Volume Summaries

## 第一卷：山野少年 (Ch.1-30)

少年张三离开山村踏上修仙之路，在青云宗入门考核中觉醒变异灵根。入门后拜入内门长老门下，与同门师兄妹建立初步关系。张三在宗门大比中崭露头角，但因灵根变异引来高层关注。同期，魔道势力暗中渗透青云宗，张三意外发现一柄古剑中的残魂，获得上古传承。宗门大比决赛中，张三击败宿敌晋升内门，却也暴露了古剑秘密，引来魔道觊觎。卷末，青云宗遭魔道突袭，张三在混战中保护师妹撤退，古剑残魂首次显威击退强敌，但张三重伤昏迷。

张三伤愈后修为大进，开始调查魔道渗透的幕后真相。在追查过程中，他发现宗门内有叛徒勾结魔道...
```

### 3.7 写入 volume_summaries.md

最终 `volume_summaries.md` 包含所有已完成卷的叙事段落，写作时作为上下文加载，替代逐章摘要，大幅减少 token 消耗。

**合并前后对比**：

| 对比项 | 合并前 | 合并后 |
|--------|--------|--------|
| chapter_summaries.md | 30 行（第一卷）+ 30 行（第二卷）+ 5 行（第三卷）= 65 行 | 5 行（仅第三卷） |
| volume_summaries.md | 空 | 第一卷段落 + 第二卷段落 ≈ 1000 字 |
| summaries_archive/ | 空 | vol_1-30.md + vol_31-60.md |
| token 消耗 | ~8000 tokens | ~1500 tokens |

## 四、ConsolidationResult 结构体

```go
// ConsolidationResult 卷级合并结果
type ConsolidationResult struct {
    // VolumeSummaries 合并后的 volume_summaries.md 全文
    // 包含所有已完成卷的叙事段落
    VolumeSummaries string

    // ArchivedVolumes 本次归档的卷数
    // = 已完成卷的数量
    ArchivedVolumes int

    // RetainedChapters 保留在 chapter_summaries.md 中的章节数
    // = 当前卷的章节数 + 不被任何卷覆盖的章节数
    RetainedChapters int

    // PromotedHookCount 本次 promoted 标志从 false 翻转为 true 的伏笔数
    // 来自 rerunAdvancedCountPromotion
    // 0 表示 pending_hooks.md 不存在或没有伏笔超过阈值
    PromotedHookCount int
}
```

**各字段的典型值**：

| 场景 | VolumeSummaries | ArchivedVolumes | RetainedChapters | PromotedHookCount |
|------|-----------------|-----------------|------------------|-------------------|
| 新书（无已完成卷） | "" | 0 | 当前章节数 | 0-N（伏笔晋升独立执行） |
| 第一卷完成 | 第一卷段落 | 1 | 第二卷章节数 | 0-N |
| 无卷边界 | "" | 0 | 全部章节数 | 0-N |
| 重复调用 | 不变 | 0 | 不变 | 0（已晋升过） |

## 五、完整 Go 接口定义

```go
package agents

import (
    "context"
    "os"
    "path/filepath"
    "regexp"
    "strconv"
    "strings"
)

// --- 结果结构体 ---

// ConsolidationResult 卷级合并结果
type ConsolidationResult struct {
    VolumeSummaries   string
    ArchivedVolumes   int
    RetainedChapters  int
    PromotedHookCount int
}

// --- 内部结构体 ---

// VolumeBoundary 卷边界
type VolumeBoundary struct {
    Name    string
    StartCh int
    EndCh   int
}

// CompletedVolume 已完成卷（含摘要行）
type CompletedVolume struct {
    Name    string
    StartCh int
    EndCh   int
    Rows    []SummaryRow
}

// SummaryRow 章节摘要表行
type SummaryRow struct {
    Chapter int    // 章节号
    Raw     string // 原始 markdown 行
}

// StoredHook 伏笔（存储格式）
type StoredHook struct {
    HookID            string
    StartChapter      int
    Type              string
    Status            string
    LastAdvancedChapter int
    ExpectedPayoff    string
    Notes             string
    Promoted          bool   // 是否已晋升
    AdvancedCount     int    // 已推进章节数
}

// PromotionResult 伏笔晋升结果
type PromotionResult struct {
    Hooks        []StoredHook
    Updated      bool
    FlippedCount int
}

// --- ConsolidatorAgent ---

// ConsolidatorAgent 卷级合并 Agent
type ConsolidatorAgent struct {
    *BaseAgent
}

// Name 返回 Agent 名称
func (a *ConsolidatorAgent) Name() string {
    return "consolidator"
}

// Consolidate 执行卷级合并。
//
// 完整步骤：
//   1. 读取 chapter_summaries.md + volume_map.md
//   2. rerunAdvancedCountPromotion（伏笔晋升，独立执行）
//   3. parseVolumeBoundaries（解析卷边界）
//   4. parseSummaryTable（解析摘要表）
//   5. 识别已完成卷（endCh <= maxChapter）
//   6. 对每个已完成卷：LLM 压缩为 ≤500字叙事段落
//   7. 归档旧摘要到 summaries_archive/
//   8. 重写 chapter_summaries.md（仅保留当前卷行）
//   9. 追加写入 volume_summaries.md
func (a *ConsolidatorAgent) Consolidate(ctx context.Context, bookDir string) (*ConsolidationResult, error)

// --- 内部方法 ---

// parseVolumeBoundaries 从 volume_map.md 解析卷边界。
//
// 匹配规则：
//   1. 行首匹配：第N卷 / Volume N（忽略 markdown # 前缀）
//   2. 行中匹配章节范围：(第1-30章) / (Chapters 1-30) / 第1-30章
//
// 返回空切片表示无法解析卷边界。
func (a *ConsolidatorAgent) parseVolumeBoundaries(outline string) []VolumeBoundary

// parseSummaryTable 解析章节摘要表。
//
// 返回：
//   - header：表头行（含列名行 + 分隔行）
//   - rows：数据行（含章节号和原始 markdown）
func (a *ConsolidatorAgent) parseSummaryTable(raw string) (header string, rows []SummaryRow)

// rerunAdvancedCountPromotion 重新运行伏笔晋升检查。
//
// 独立于摘要合并执行，即使没有已完成卷也会运行。
// 原因：新书可能没有已完成卷，但伏笔仍需晋升；
//       归档后摘要表变短可能影响 advancedCount 计算。
//
// 返回：本次翻转的伏笔数量
func (a *ConsolidatorAgent) rerunAdvancedCountPromotion(storyDir string) (int, error)

// --- 辅助函数 ---

// rerunPromotionPass 检查所有伏笔的 advancedCount 是否超过阈值（默认2章）。
//
// advancedCount 计算：扫描 chapter_summaries.md，
// 统计每个伏笔在摘要中被提及/推进的章节数。
//
// 晋升条件：advancedCount >= 2 且当前 promoted=false
func rerunPromotionPass(hooks []StoredHook, summariesRaw string) PromotionResult

// parsePendingHooksMarkdown 解析 pending_hooks.md 为伏笔列表
func parsePendingHooksMarkdown(raw string) []StoredHook

// renderHookSnapshot 将伏笔列表渲染为 markdown 表格
func renderHookSnapshot(hooks []StoredHook, language string) string

// readVolumeMap 读取 volume_map.md（fallback: volume_outline.md）
func readVolumeMap(bookDir string) (string, error)

// detectLanguage 检测文本语言（包含中文字符则返回 "zh"，否则 "en"）
func detectLanguage(text string) string

// --- BaseAgent 接口（ConsolidatorAgent 依赖）---

// BaseAgent 基础 Agent
type BaseAgent struct {
    // ... LLM 客户端、项目根目录等
}

// Chat 调用 LLM
func (a *BaseAgent) Chat(ctx context.Context, messages []Message, options ChatOptions) (*ChatResponse, error)

// Message LLM 消息
type Message struct {
    Role    string // "system" / "user" / "assistant"
    Content string
}

// ChatOptions LLM 调用选项
type ChatOptions struct {
    Temperature float64
    MaxTokens   int
}

// ChatResponse LLM 响应
type ChatResponse struct {
    Content string
    Usage   TokenUsage
}

// TokenUsage token 用量
type TokenUsage struct {
    PromptTokens     int
    CompletionTokens int
    TotalTokens      int
}
```

## 附录：完整流程时序

```
Consolidate(bookDir)
  │
  ├─ 读取 chapter_summaries.md + volume_map.md（并行）
  │
  ├─ rerunAdvancedCountPromotion(storyDir)          ← 伏笔晋升（独立执行）
  │   ├─ 读取 pending_hooks.md
  │   ├─ parsePendingHooksMarkdown → hooks
  │   ├─ 读取 chapter_summaries.md（用于计算 advancedCount）
  │   ├─ rerunPromotionPass(hooks, summariesRaw)
  │   │   └─ 对每个 hook: advancedCount >= 2 → promoted=true
  │   └─ 写回 pending_hooks.md
  │
  ├─ if summariesRaw == "" || outlineRaw == "":     ← 任一文件缺失
  │   └─ return { promotedHookCount }               ← 伏笔晋升已执行
  │
  ├─ parseVolumeBoundaries(outlineRaw)              ← 解析卷边界
  │   └─ 匹配 "第N卷" + "(第M-K章)"
  │
  ├─ if len(volumeBoundaries) == 0:                 ← 无卷边界
  │   └─ return { promotedHookCount }
  │
  ├─ parseSummaryTable(summariesRaw)                ← 解析摘要表
  │   └─ header + rows
  │
  ├─ if len(rows) == 0:                             ← 无摘要行
  │   └─ return { promotedHookCount }
  │
  ├─ maxChapter = max(rows[].Chapter)
  │
  ├─ 分类卷：                                       ← 识别已完成卷
  │   ├─ completedVolumes: endCh <= maxChapter 且有行
  │   └─ currentVolumeRows: 其余行（当前卷 + 未覆盖行）
  │
  ├─ if len(completedVolumes) == 0:                 ← 无已完成卷
  │   └─ return { retainedChapters: len(currentVolumeRows), promotedHookCount }
  │
  ├─ 读取已有 volume_summaries.md
  │
  ├─ for vol in completedVolumes:                   ← LLM 压缩
  │   ├─ 拼接 vol.Rows
  │   ├─ Chat(system=summarizer, user=vol+rows, temp=0.3)
  │   └─ 追加 "## vol.Name (Ch.start-end)\n\nresponse"
  │
  ├─ 写入 volume_summaries.md                       ← 追加叙事段落
  │
  ├─ for vol in completedVolumes:                   ← 归档
  │   └─ 写入 summaries_archive/vol_start-end.md
  │
  ├─ 重写 chapter_summaries.md                      ← 仅保留当前卷行
  │   └─ header + currentVolumeRows
  │
  └─ return ConsolidationResult{
         VolumeSummaries:   newSummaries,
         ArchivedVolumes:   len(completedVolumes),
         RetainedChapters:  len(currentVolumeRows),
         PromotedHookCount: promotedHookCount,
       }
```
