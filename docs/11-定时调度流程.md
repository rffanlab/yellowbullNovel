## 一、流程概述

Scheduler 是自动写作的核心调度器，负责定时执行写作循环和扫榜扫描。它管理多本书的并发写作，通过质量门机制在章节连续失败时自动暂停书籍，防止持续产出低质量内容。

```
Scheduler.start()
  │
  ├─ 立即触发一次 runWriteCycle                       ← 启动即写
  │
  ├─ 定时任务1：write-cycle（按 cron 间隔）
  │   └─ runWriteCycle
  │       ├─ isDailyCapReached? → 跳过
  │       ├─ listBooks → 过滤 active/outlining
  │       ├─ slice(maxConcurrentBooks)
  │       └─ 并行 processBook
  │           └─ for i in chaptersPerCycle:
  │               ├─ cooldown（章节间冷却）
  │               ├─ writeOneChapter
  │               │   ├─ PipelineRunner.writeNextChapter
  │               │   ├─ if ready-for-review:
  │               │   │   ├─ 清零 consecutiveFailures
  │               │   │   ├─ recordChapterWritten
  │               │   │   ├─ if detection.enabled: runDetection
  │               │   │   └─ onChapterComplete 通知
  │               │   └─ else (audit-failed):
  │               │       ├─ handleAuditFailure
  │               │       │   ├─ consecutiveFailures++
  │               │       │   ├─ failureDimensions 聚类
  │               │       │   ├─ if dim.count >= 3: emitDiagnosticAlert
  │               │       │   └─ if failures >= threshold: pauseBook
  │               │       └─ onChapterComplete 通知
  │               └─ if failed && retry: 重试一次
  │
  └─ 定时任务2：radar-scan（按 cron 间隔）
      └─ PipelineRunner.runRadar()
```

**核心设计原则**：
1. **启动即写**：`start()` 立即触发一次写作循环，不等第一个定时周期
2. **防重叠**：写作循环和扫榜各自用 inFlight 标记防止重叠执行
3. **并行多书**：每个写作循环并行处理 `maxConcurrentBooks` 本书
4. **质量门暂停**：连续失败超阈值自动暂停书籍，需要手动 resumeBook
5. **日章数上限**：全局每日写作章数上限，防止 LLM 费用失控

## 三、Scheduler 设计

### 3.1 start()：立即触发 + 定时任务

```go
func (s *Scheduler) Start(ctx context.Context) error {
    if s.running {
        return nil // 幂等
    }
    s.running = true

    // 1. 立即触发一次写作循环
    s.triggerWriteCycle(ctx)

    // 2. 启动定时任务1：write-cycle
    writeCycleMs := s.cronToMs(s.config.WriteCron)
    s.writeTicker = time.NewTicker(time.Duration(writeCycleMs) * time.Millisecond)
    go func() {
        for {
            select {
            case <-s.writeTicker.C:
                if err := s.triggerWriteCycle(ctx); err != nil {
                    s.config.OnError("scheduler", err)
                }
            case <-s.stopCh:
                return
            }
        }
    }()

    // 3. 启动定时任务2：radar-scan
    radarMs := s.cronToMs(s.config.RadarCron)
    s.radarTicker = time.NewTicker(time.Duration(radarMs) * time.Millisecond)
    go func() {
        for {
            select {
            case <-s.radarTicker.C:
                if err := s.triggerRadarScan(ctx); err != nil {
                    s.config.OnError("radar", err)
                }
            case <-s.stopCh:
                return
            }
        }
    }()

    return nil
}

func (s *Scheduler) Stop() {
    s.running = false
    if s.writeTicker != nil {
        s.writeTicker.Stop()
    }
    if s.radarTicker != nil {
        s.radarTicker.Stop()
    }
    close(s.stopCh)
}
```

**cronToMs 简化解析**（inkos 的 cron 只支持简单格式）：

```go
// cronToMs 将简化的 cron 表达式转为毫秒间隔。
//
// 支持格式：
//   "*/N * * * *" → 每 N 分钟（N * 60 * 1000 ms）
//   "0 */N * * *" → 每 N 小时（N * 60 * 60 * 1000 ms）
//   其他          → 每天（24 * 60 * 60 * 1000 ms）
func (s *Scheduler) cronToMs(cron string) int64 {
    parts := strings.Fields(cron)
    if len(parts) < 5 {
        return 24 * 60 * 60 * 1000
    }
    minute := parts[0]
    hour := parts[1]

    // "*/N * * * *" → 每 N 分钟
    if strings.HasPrefix(minute, "*/") {
        n, _ := strconv.Atoi(minute[2:])
        return int64(n) * 60 * 1000
    }
    // "0 */N * * *" → 每 N 小时
    if strings.HasPrefix(hour, "*/") {
        n, _ := strconv.Atoi(hour[2:])
        return int64(n) * 60 * 60 * 1000
    }
    // 固定时间 → 每天
    return 24 * 60 * 60 * 1000
}
```

**Go 实现建议**：生产环境可用 `github.com/robfig/cron/v3` 库替代简化解析，支持完整 cron 语法。

### 3.2 定时任务1：write-cycle

```go
// triggerWriteCycle 触发写作循环（防重叠）。
func (s *Scheduler) triggerWriteCycle(ctx context.Context) error {
    // 防重叠：如果上一轮还在跑，跳过本次
    if s.writeCycleInFlight {
        s.log.Warn("Write cycle still running, skipping overlapping tick")
        return nil
    }

    s.writeCycleInFlight = true
    defer func() { s.writeCycleInFlight = false }()

    return s.runWriteCycle(ctx)
}
```

**防重叠机制**：
- 用 `writeCycleInFlight` 布尔值标记当前是否有写作循环在执行
- 如果定时器触发时上一轮还没结束，直接跳过本次
- 避免多个写作循环并发导致书籍锁竞争和资源浪费

### 3.3 定时任务2：radar-scan

```go
// triggerRadarScan 触发扫榜（防重叠）。

}

// runRadarScan 执行扫榜。

    return s.pipeline.RunRadar(ctx)
```

### 3.4 runWriteCycle：遍历活跃书籍

```go
// runWriteCycle 执行一次写作循环。
func (s *Scheduler) runWriteCycle(ctx context.Context) error {
    // 1. 检查日章数上限
    if s.isDailyCapReached() {
        s.log.Info("Daily cap reached (%d), skipping cycle", s.config.MaxChaptersPerDay)
        return nil
    }

    // 2. 获取所有书籍
    bookIDs := s.state.ListBooks()

    // 3. 过滤活跃书籍（active / outlining），排除暂停的书
    var activeBooks []BookEntry
    for _, id := range bookIDs {
        if s.pausedBooks[id] {
            continue
        }
        config, err := s.state.LoadBookConfig(id)
        if err != nil {
            continue
        }
        if config.Status == "active" || config.Status == "outlining" {
            activeBooks = append(activeBooks, BookEntry{ID: id, Config: config})
        }
    }

    // 4. 限制并发数
    booksToWrite := activeBooks
    if len(booksToWrite) > s.config.MaxConcurrentBooks {
        booksToWrite = booksToWrite[:s.config.MaxConcurrentBooks]
    }

    // 5. 并行处理多本书
    var wg sync.WaitGroup
    for _, book := range booksToWrite {
        wg.Add(1)
        go func(b BookEntry) {
            defer wg.Done()
            s.processBook(ctx, b.ID, b.Config)
        }(book)
    }
    wg.Wait()

    return nil
}
```

### 3.5 processBook：单书多章写作

```go
// processBook 处理单本书：写 chaptersPerCycle 章带重试+冷却。
func (s *Scheduler) processBook(ctx context.Context, bookID string, bookConfig *BookConfig) {
    for i := 0; i < s.config.ChaptersPerCycle; i++ {
        // 检查停止条件
        if !s.running {
            return
        }
        if s.isDailyCapReached() {
            return
        }
        if s.pausedBooks[bookID] {
            return
        }

        // 章节间冷却（第一章不冷却）
        if i > 0 && s.config.CooldownAfterChapterMs > 0 {
            time.Sleep(time.Duration(s.config.CooldownAfterChapterMs) * time.Millisecond)
        }

        // 写一章
        success := s.writeOneChapter(ctx, bookID, bookConfig)
        if !success {
            // 失败重试（如果重试次数未超限）
            failures := s.consecutiveFailures[bookID]
            if failures <= s.gates.MaxAuditRetries && s.config.RetryDelayMs > 0 {
                s.log.Warn("%s retrying in %dms", bookID, s.config.RetryDelayMs)
                time.Sleep(time.Duration(s.config.RetryDelayMs) * time.Millisecond)
                retrySuccess := s.writeOneChapter(ctx, bookID, bookConfig)
                if !retrySuccess {
                    break // 二次失败，停止本书的循环
                }
            } else {
                break // 超过重试限制，停止本书的循环
            }
        }
    }
}
```

**停止条件**（每章循环前检查）：
1. 调度器已停止（`!s.running`）
2. 日章数已达上限（`isDailyCapReached`）
3. 书被暂停（`pausedBooks[bookID]`）

### 3.6 writeOneChapter：写一章 + 检测

```go
// writeOneChapter 为一本书写一章。返回 true 表示审查通过。
func (s *Scheduler) writeOneChapter(ctx context.Context, bookID string, bookConfig *BookConfig) bool {
    // 1. 计算温度覆盖（失败次数越多温度越高，增加创造性）
    failures := s.consecutiveFailures[bookID]
    var tempOverride *float64
    if failures > 0 {
        temp := math.Min(1.2, 0.7+float64(failures)*s.gates.RetryTemperatureStep)
        tempOverride = &temp
    }

    // 2. 调用 PipelineRunner 写下一章
    result, err := s.pipeline.WriteNextChapter(ctx, bookID, "", tempOverride)
    if err != nil {
        s.config.OnError(bookID, err)
        s.handleAuditFailure(ctx, bookID, 0, nil)
        return false
    }

    // 3. 审查通过
    if result.Status == "ready-for-review" {
        // 清零连续失败计数
        delete(s.consecutiveFailures, bookID)

        // 记录日章数
        s.recordChapterWritten()

        // 自动检测（如果启用）
        if s.config.Detection != nil && s.config.Detection.Enabled {
            s.runDetection(ctx, bookID, bookConfig, result.ChapterNumber)
        }

        // 通知
        if s.config.OnChapterComplete != nil {
            s.config.OnChapterComplete(bookID, result.ChapterNumber, result.Status)
        }
        return true
    }

    // 4. 审查失败 — 应用质量门
    issueCategories := make([]string, len(result.AuditResult.Issues))
    for i, issue := range result.AuditResult.Issues {
        issueCategories[i] = issue.Category
    }
    s.handleAuditFailure(ctx, bookID, result.ChapterNumber, issueCategories)

    if s.config.OnChapterComplete != nil {
        s.config.OnChapterComplete(bookID, result.ChapterNumber, result.Status)
    }
    return false
}
```

**温度递增策略**：

| 失败次数 | 温度 | 说明 |
|----------|------|------|
| 0 | 0.7（默认） | 正常写作 |
| 1 | 0.8 | 0.7 + 1×0.1 |
| 2 | 0.9 | 0.7 + 2×0.1 |
| 3 | 1.0 | 0.7 + 3×0.1 |
| 4 | 1.1 | 0.7 + 4×0.1 |
| 5+ | 1.2（封顶） | 0.7 + 5×0.1，但 min(1.2, ...) |

**runDetection 检测循环**：

```go
// runDetection 执行章节检测 + 可选自动重写。
func (s *Scheduler) runDetection(ctx context.Context, bookID string, bookConfig *BookConfig, chapterNumber int) {
    if s.config.Detection == nil {
        return
    }

    bookDir := s.state.BookDir(bookID)
    chapterContent, err := s.readChapterContent(bookDir, chapterNumber)
    if err != nil {
        s.config.OnError(bookID, err)
        return
    }

    // 检测
    detResult, err := detectChapter(ctx, s.config.Detection, chapterContent, chapterNumber)
    if err != nil {
        s.config.OnError(bookID, err)
        return
    }

    // 未通过 + 启用自动重写
    if !detResult.Passed && s.config.Detection.AutoRewrite {
        _, err := detectAndRewrite(ctx, s.config.Detection,
            s.config.Client, s.config.Model, s.config.ProjectRoot,
            bookDir, chapterContent, chapterNumber, bookConfig.Genre)
        if err != nil {
            s.config.OnError(bookID, err)
        }
    }
}
```

## 四、质量门机制

### 4.1 consecutiveFailures 连续失败次数

每本书维护一个连续失败计数器，审查通过时清零，审查失败时递增。

```go
// consecutiveFailures: bookID → 连续失败次数
consecutiveFailures map[string]int
```

**行为**：
- 审查通过（`ready-for-review`）：`delete(consecutiveFailures, bookID)` — 清零
- 审查失败（`audit-failed`）：`consecutiveFailures[bookID]++`
- 超过阈值（`pauseAfterConsecutiveFailures`，默认3）：暂停书籍

```go
// handleAuditFailure 处理审查失败。
func (s *Scheduler) handleAuditFailure(ctx context.Context, bookID string, chapterNumber int, issueCategories []string) {
    failures := s.consecutiveFailures[bookID] + 1
    s.consecutiveFailures[bookID] = failures

    // 维度聚类
    if len(issueCategories) > 0 {
        dimMap := s.failureDimensions[bookID]
        if dimMap == nil {
            dimMap = make(map[string]int)
        }
        for _, cat := range issueCategories {
            dimMap[cat]++
        }
        s.failureDimensions[bookID] = dimMap

        // 检查维度聚类（任一维度 >= 3 次失败）
        for dim, count := range dimMap {
            if count >= 3 {
                s.emitDiagnosticAlert(ctx, bookID, chapterNumber, dim, count)
            }
        }
    }

    gates := s.gates()

    // 未超重试限制：只记录，不暂停
    if failures <= gates.MaxAuditRetries {
        s.log.Warn("%s audit failed (%d/%d), will retry", bookID, failures, gates.MaxAuditRetries)
        return
    }

    // 超过暂停阈值：暂停书籍
    if failures >= gates.PauseAfterConsecutiveFailures {
        s.pausedBooks[bookID] = true
        reason := fmt.Sprintf("%d consecutive audit failures (threshold: %d)",
            failures, gates.PauseAfterConsecutiveFailures)
        s.log.Error("%s PAUSED: %s", bookID, reason)

        if s.config.OnPause != nil {
            s.config.OnPause(bookID, reason)
        }

        // 发送通知
        if len(s.config.NotifyChannels) > 0 {
            dispatchWebhookEvent(ctx, s.config.NotifyChannels, WebhookEvent{
                Event:         "pipeline-error",
                BookID:        bookID,
                ChapterNumber: chapterNumber,
                Timestamp:     time.Now().UTC().Format(time.RFC3339),
                Data: map[string]interface{}{
                    "reason":             reason,
                    "consecutiveFailures": failures,
                },
            })
        }
    }
}
```

### 4.2 failureDimensions 失败维度聚类

跟踪每本书在各失败维度上的累计次数，当某维度累计 ≥3 次时触发诊断告警。

```go
// failureDimensions: bookID → (dimension → count)
failureDimensions map[string]map[string]int
```

**维度示例**（来自 AuditIssue.Category）：
- `Character Fidelity` — 人设崩坏
- `Mainline Drift` — 主线偏离
- `Timeline` — 时间线错误
- `Hook Check` — 伏笔检查
- `Pacing` — 节奏问题
- `POV Consistency` — 视角一致性

**诊断告警**：

```go
// emitDiagnosticAlert 发送维度聚类告警。
func (s *Scheduler) emitDiagnosticAlert(ctx context.Context, bookID string, chapterNumber int, dimension string, count int) {
    s.log.Warn("DIAGNOSTIC: %s has %d failures in dimension %q", bookID, count, dimension)

    if len(s.config.NotifyChannels) > 0 {
        dispatchWebhookEvent(ctx, s.config.NotifyChannels, WebhookEvent{
            Event:         "diagnostic-alert",
            BookID:        bookID,
            ChapterNumber: chapterNumber,
            Timestamp:     time.Now().UTC().Format(time.RFC3339),
            Data: map[string]interface{}{
                "dimension":    dimension,
                "failureCount": count,
            },
        })
    }
}
```

**用途**：维度聚类告警帮助发现系统性问题。例如某本书连续3次在 `Character Fidelity` 维度失败，说明人设层面有根本性问题，可能需要回炉重写而非继续写新章。

### 4.3 dailyChapterCount 日章数上限

全局每日写作章数计数器，防止 LLM 费用失控。

```go
// dailyChapterCount: "YYYY-MM-DD" → count
dailyChapterCount map[string]int
```

```go
// isDailyCapReached 检查日章数是否已达上限。
func (s *Scheduler) isDailyCapReached() bool {
    today := time.Now().UTC().Format("2006-01-02")
    count := s.dailyChapterCount[today]
    return count >= s.config.MaxChaptersPerDay
}

// recordChapterWritten 记录一章写作完成。
func (s *Scheduler) recordChapterWritten() {
    today := time.Now().UTC().Format("2006-01-02")
    s.dailyChapterCount[today]++

    // 清理旧日期（只保留今天）
    for key := range s.dailyChapterCount {
        if key != today {
            delete(s.dailyChapterCount, key)
        }
    }
}
```

**检查时机**：
- `runWriteCycle` 开始时
- `processBook` 每章循环前

### 4.4 pausedBooks 暂停的书

```go
// pausedBooks: bookID → bool
pausedBooks map[string]bool
```

**暂停触发条件**：
- `consecutiveFailures >= pauseAfterConsecutiveFailures`（默认3）

**暂停效果**：
- `runWriteCycle` 跳过暂停的书
- `processBook` 检测到暂停后立即停止

**恢复**：

```go
// ResumeBook 恢复暂停的书籍。
func (s *Scheduler) ResumeBook(bookID string) {
    delete(s.pausedBooks, bookID)
    delete(s.consecutiveFailures, bookID)
    delete(s.failureDimensions, bookID)
}

// IsBookPaused 检查书籍是否被暂停。
func (s *Scheduler) IsBookPaused(bookID string) bool {
    return s.pausedBooks[bookID]
}
```

**恢复时清零所有计数器**：连续失败数、维度聚类、暂停标记全部清除，让书籍从头开始。

### 4.5 cooldownAfterChapterMs 章节间冷却

同一本书连续写多章时，每章之间等待一段时间，避免：
1. LLM API 速率限制
2. 上下文窗口碎片化（truth files 刚写入，需要时间稳定）
3. 费用集中爆发

```go
// processBook 中的冷却逻辑
for i := 0; i < s.config.ChaptersPerCycle; i++ {
    // ...
    if i > 0 && s.config.CooldownAfterChapterMs > 0 {
        time.Sleep(time.Duration(s.config.CooldownAfterChapterMs) * time.Millisecond)
    }
    // writeOneChapter...
}
```

**典型配置值**：
- `cooldownAfterChapterMs = 30000`（30秒）— 适合个人创作
- `cooldownAfterChapterMs = 300000`（5分钟）— 适合批量生产

**QualityGates 默认值**：

```go
func (s *Scheduler) gates() *QualityGates {
    if s.config.QualityGates != nil {
        return s.config.QualityGates
    }
    return &QualityGates{
        MaxAuditRetries:               2,   // 最多重试2次
        PauseAfterConsecutiveFailures: 3,   // 连续失败3次暂停
        RetryTemperatureStep:          0.1, // 每次失败温度+0.1
    }
}
```

## 五、通知机制

Scheduler 支持三种通知回调 + Webhook 推送：

### 5.1 回调通知

```go
// SchedulerConfig 中的回调
type SchedulerConfig struct {
    // OnChapterComplete 章节写作完成通知（无论成功/失败）
    OnChapterComplete func(bookID string, chapter int, status string)

    // onError 错误通知
    OnError func(bookID string, err error)

    // OnPause 书籍暂停通知
    OnPause func(bookID string, reason string)
}
```

**触发时机**：

| 回调 | 触发时机 | 参数 |
|------|----------|------|
| `OnChapterComplete` | 每章写完后（无论 ready-for-review / audit-failed） | bookID, chapterNumber, status |
| `OnError` | 写章异常、检测异常、扫榜异常 | bookID（或 "scheduler"/"radar"）, error |
| `OnPause` | 书籍被质量门暂停 | bookID, reason |

### 5.2 Webhook 推送

除了回调，还支持通过 `NotifyChannels` 推送 Webhook 事件：

```go
// Webhook 事件类型
const (
    WebhookEventPipelineError    = "pipeline-error"    // 书籍暂停
    WebhookEventDiagnosticAlert  = "diagnostic-alert"  // 维度聚类告警
)

// WebhookEvent Webhook 事件
type WebhookEvent struct {
    Event         string                 // 事件类型
    BookID        string
    ChapterNumber int
    Timestamp     string                 // RFC3339
    Data          map[string]interface{} // 事件数据
}
```

**pipeline-error 事件**（书籍暂停时）：
```json
{
  "event": "pipeline-error",
  "bookId": "book-001",
  "chapterNumber": 45,
  "timestamp": "2025-07-11T12:00:00Z",
  "data": {
    "reason": "3 consecutive audit failures (threshold: 3)",
    "consecutiveFailures": 3
  }
}
```

**diagnostic-alert 事件**（维度聚类时）：
```json
{
  "event": "diagnostic-alert",
  "bookId": "book-001",
  "chapterNumber": 45,
  "timestamp": "2025-07-11T12:00:00Z",
  "data": {
    "dimension": "Character Fidelity",
    "failureCount": 3
  }
}
```

## 附录：完整流程时序

```
Scheduler.Start()
  │
  ├─ if running: return                              ← 幂等
  ├─ running = true
  │
  ├─ triggerWriteCycle()                             ← 启动即写
  │   └─ runWriteCycle()
  │       ├─ isDailyCapReached? → skip
  │       ├─ listBooks → 过滤 active/outlining → 排除 paused
  │       ├─ slice(maxConcurrentBooks)
  │       └─ 并行 processBook(book)
  │           └─ for i in chaptersPerCycle:
  │               ├─ 检查停止条件（!running / dailyCap / paused）
  │               ├─ if i > 0: sleep(cooldownAfterChapterMs)
  │               ├─ writeOneChapter(book)
  │               │   ├─ tempOverride = 0.7 + failures × 0.1（封顶1.2）
  │               │   ├─ pipeline.WriteNextChapter(bookID, tempOverride)
  │               │   ├─ if ready-for-review:
  │               │   │   ├─ 清零 consecutiveFailures
  │               │   │   ├─ recordChapterWritten
  │               │   │   ├─ if detection.enabled: runDetection
  │               │   │   │   └─ if !passed && autoRewrite: detectAndRewrite
  │               │   │   └─ OnChapterComplete(bookID, ch, "ready-for-review")
  │               │   └─ if audit-failed:
  │               │       ├─ handleAuditFailure
  │               │       │   ├─ consecutiveFailures++
  │               │       │   ├─ failureDimensions 聚类
  │               │       │   ├─ if dim.count >= 3: emitDiagnosticAlert
  │               │       │   └─ if failures >= 3: pauseBook + OnPause + Webhook
  │               │       ├─ OnChapterComplete(bookID, ch, "audit-failed")
  │               │       └─ return false
  │               └─ if !success && failures <= maxRetries:
  │                   ├─ sleep(retryDelayMs)
  │                   └─ retry writeOneChapter
  │
  ├─ 启动 writeTicker（按 writeCron 间隔）
  │   └─ 每个 tick: triggerWriteCycle()（防重叠）
  │
  └─ 启动 radarTicker（按 radarCron 间隔）
      └─ 每个 tick: triggerRadarScan()（防重叠）
          └─ pipeline.RunRadar()

Scheduler.Stop()
  ├─ running = false
  ├─ writeTicker.Stop()
  ├─ radarTicker.Stop()
  └─ close(stopCh)

Scheduler.ResumeBook(bookID)
  ├─ delete(pausedBooks, bookID)
  ├─ delete(consecutiveFailures, bookID)
  └─ delete(failureDimensions, bookID)
```
