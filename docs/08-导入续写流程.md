## 一、流程概述

导入续写用于把已有正文导入系统，让 Writer 和 Auditor 能自然续写后续章节。核心难点是：已有正文没有 truth files（状态卡、伏笔池、角色矩阵等），必须从正文反向推导出来。

inkos 采用**两阶段策略**：

```
已有正文（N 章）
  │
  ▼
┌─────────────────────────────────────────────────────┐
│ Step 1: 构建导入资料包 + 反向推导基础设定           │
│                                                      │
│  buildImportFoundationSource(chapters)               │
│    ├─ 小书(≤80K字)：全量打包                         │
│    └─ 大书(>80K字)：压缩打包                         │
│        ├─ 开头4章 + 结尾4章（完整）                  │
│        ├─ 中段8个锚点（头尾各摘录6000字）            │
│        └─ 章节标题目录（截断到24K字）                │
│                                                      │
│  ArchitectAgent.generateFoundationFromImport         │
│    → story_frame / volume_map / roles /              │
│      book_rules / pending_hooks                      │
│                                                      │
│  FoundationReviewerAgent.review（series 模式才走）   │
│    → 通过 / 打回重生成（最多2轮）                    │
│                                                      │
│  写入 foundation files + 重置 truth files 到种子态   │
└────────────────────────┬────────────────────────────┘
                         ▼
┌─────────────────────────────────────────────────────┐
│ Step 2: 逐章回放重建 truth files                     │
│                                                      │
│  for chapter 1..N:                                   │
│    prepareWriteInput（构建 governed 上下文）          │
│    ChapterAnalyzerAgent.analyzeChapter               │
│      → 用已有正文作为 content，让 LLM 提取：          │
│        - 章节标题/摘要                               │
│        - 状态卡更新                                   │
│        - 伏笔池更新                                   │
│        - 子线/情感弧/角色矩阵                         │
│    WriterAgent.saveChapter（保存章节文件）            │
│    WriterAgent.saveNewTruthFiles（保存扩展 truth）    │
│    saveChapterIndex（更新章节索引）                   │
│    snapshotState（快照，支持回滚+续跑）               │
└────────────────────────┬────────────────────────────┘
                         ▼
                 ImportChaptersResult
  { bookId, importedCount, totalWords, nextChapter }
```

**核心设计原则**：
1. **资料包压缩**：大书不把全部正文塞给 LLM，而是用锚点+目录让 LLM 理解全局，逐章回放时才用完整正文
2. **WriterAgent 复用**：回放阶段不调用 WriterAgent 生成正文（正文已有），只用它保存章节和 truth files
3. **ChapterAnalyzerAgent 是核心**：它读取已有正文，像 Writer 一样提取 truth file 更新，但 content 用原文
4. **每章快照**：每章回放后做一次 snapshotState，支持从任意章节断点续跑

## 三、Step 1：构建导入资料包

### 3.1 小书全量打包（≤80K 字）

当全部章节总字数 ≤ 80000 字时，直接全量打包，不做压缩：

```go
// 全量打包：每章格式化为 "第N章 标题\n\n正文"，章节间用 "\n\n---\n\n" 分隔
func formatImportedChapter(chapter ImportedChapter, index int, language LengthLanguage) string {
    if language == LanguageEn {
        return fmt.Sprintf("Chapter %d: %s\n\n%s", index+1, chapter.Title, chapter.Content)
    }
    return fmt.Sprintf("第%d章 %s\n\n%s", index+1, chapter.Title, chapter.Content)
}

// 估算总字数：每章 = len(title) + len(content) + 24（格式开销）
func estimateImportFullTextLength(chapters []ImportedChapter) int {
    total := 0
    for _, ch := range chapters {
        total += len(ch.Title) + len(ch.Content) + 24
    }
    return total
}
```

**全量打包逻辑**：
```go
if estimateImportFullTextLength(chapters) <= 80000 {
    parts := make([]string, len(chapters))
    for i, ch := range chapters {
        parts[i] = formatImportedChapter(ch, i, language)
    }
    return strings.Join(parts, "\n\n---\n\n")
}
```

### 3.2 大书压缩打包（>80K 字）

超过 80K 字时，采用压缩策略，保留：

| 组成部分 | 数量 | 内容 | 目的 |
|----------|------|------|------|
| 开头章节 | 4 章 | 完整正文 | 理解开篇设定、人设、世界观 |
| 结尾章节 | 4 章 | 完整正文 | 理解当前剧情走向、续写起点 |
| 中段锚点 | 8 章 | 头尾各摘录 6000 字 | 覆盖中段剧情发展 |
| 标题目录 | 全部 | 截断到 24K 字 | 让 LLM 知道所有章节标题和字数 |

**压缩包结构**：

```markdown
## 导入基础设定压缩资料包

本次导入共 N 章。为避免超出 LLM 上下文，这里保留开篇、结尾续写点、少量中段锚点和标题目录；完整章节将在后续顺序回放中逐章分析并沉淀 truth files。

## 章节标题目录（截断）
- 第1章：xxx（3200字）
- 第2章：xxx（2800字）
...
- ……中间 M 个章节标题省略……
...
- 第N章：xxx（3100字）

## 用于反推基础设定的正文摘录
第1章 xxx
（完整正文）

---

第4章 xxx
（完整正文）

---

第K章 xxx
（开头3600字）

【中间省略 1200 字，用于控制导入上下文预算】
（结尾2400字）

---

第N-3章 xxx
（完整正文）

---

第N章 xxx
（完整正文）
```

### 3.3 锚点选取算法

```go
// pickImportAnchorIndexes 选取导入锚点章节索引（0-based）。
//   - edgeChapterCount：开头和结尾各取的章节数（默认 4）
//   - middleAnchorCount：中段选取的锚点数（默认 8）
func pickImportAnchorIndexes(chapterCount, edgeChapterCount, middleAnchorCount int) []int {
    selected := make(map[int]struct{})

    // 开头 edgeChapterCount 章
    for i := 0; i < min(edgeChapterCount, chapterCount); i++ {
        selected[i] = struct{}{}
    }
    // 结尾 edgeChapterCount 章
    for i := max(0, chapterCount-edgeChapterCount); i < chapterCount; i++ {
        selected[i] = struct{}{}
    }

    // 中段锚点：在 [edgeChapterCount, chapterCount-edgeChapterCount) 区间均匀分布
    middleStart := min(edgeChapterCount, chapterCount)
    middleEnd := max(middleStart, chapterCount-edgeChapterCount)
    middleSize := middleEnd - middleStart
    anchors := min(middleAnchorCount, middleSize)
    for i := 0; i < anchors; i++ {
        offset := (middleSize * (i + 1)) / (anchors + 1)
        selected[min(chapterCount-1, middleStart+offset)] = struct{}{}
    }

    // 排序输出
    result := make([]int, 0, len(selected))
    for idx := range selected {
        result = append(result, idx)
    }
    sort.Ints(result)
    return result
}
```

**中段锚点均匀分布示意**（假设 middleSize=100，anchors=8）：

```
中段区间 [4, 96)
锚点位置 = 4 + (100 * (i+1)) / 9
  i=0: 4 + 11 = 15
  i=1: 4 + 22 = 26
  i=2: 4 + 33 = 37
  i=3: 4 + 44 = 48
  i=4: 4 + 55 = 59
  i=5: 4 + 66 = 70
  i=6: 4 + 77 = 81
  i=7: 4 + 88 = 92
```

### 3.4 章节标题目录截断

```go
// buildTitleCatalog 构建章节标题目录，超长时头尾保留、中间省略。
func buildTitleCatalog(chapters []ImportedChapter, language LengthLanguage, maxChars int) string {
    lines := make([]string, len(chapters))
    for i, ch := range chapters {
        if language == LanguageEn {
            lines[i] = fmt.Sprintf("- Chapter %d: %s (%d chars)", i+1, ch.Title, len(ch.Content))
        } else {
            lines[i] = fmt.Sprintf("- 第%d章：%s（%d字）", i+1, ch.Title, len(ch.Content))
        }
    }

    joined := strings.Join(lines, "\n")
    if len(joined) <= maxChars {
        return joined
    }

    // 头尾各保留 55% / 45% 预算
    headBudget := maxChars * 55 / 100
    tailBudget := maxChars - headBudget

    var head, tail []string
    headChars, tailChars := 0, 0
    for _, line := range lines {
        if headChars+len(line)+1 > headBudget {
            break
        }
        head = append(head, line)
        headChars += len(line) + 1
    }
    for i := len(lines) - 1; i >= 0; i-- {
        if tailChars+len(lines[i])+1 > tailBudget {
            break
        }
        tail = append([]string{lines[i]}, tail...)
        tailChars += len(lines[i]) + 1
    }

    omitted := len(lines) - len(head) - len(tail)
    var marker string
    if language == LanguageEn {
        marker = fmt.Sprintf("- ... %d chapter titles omitted ...", omitted)
    } else {
        marker = fmt.Sprintf("- ……中间 %d 个章节标题省略……", omitted)
    }
    return strings.Join(append(append(head, marker), tail...), "\n")
}
```

**章节头尾摘录**（用于中段锚点）：

```go
// excerptHeadTail 取正文头尾，中间用省略标记替代。
//   - maxChars=6000，头部占 60%（3600字），尾部占 40%（2400字）
func excerptHeadTail(text string, maxChars int, language LengthLanguage) string {
    clean := strings.TrimSpace(text)
    if len(clean) <= maxChars {
        return clean
    }
    headChars := max(200, maxChars*60/100)
    tailChars := max(200, maxChars-headChars)
    omitted := len(clean) - headChars - tailChars

    var marker string
    if language == LanguageEn {
        marker = fmt.Sprintf("\n\n[... %d chars omitted for import-context budget ...]\n\n", omitted)
    } else {
        marker = fmt.Sprintf("\n\n【中间省略 %d 字，用于控制导入上下文预算】\n\n", omitted)
    }
    head := strings.TrimRight(clean[:headChars], " \t\r\n")
    tail := strings.TrimLeft(clean[len(clean)-tailChars:], " \t\r\n")
    return head + marker + tail
}
```

## 四、Step 2：反向推导基础设定

### 4.1 ArchitectAgent.GenerateFoundationFromImport

从资料包反向推导 5 段基础设定：

```go
// GenerateFoundationFromImport 从已有正文资料包反向推导基础设定。
//
// 输出 5 个 section（合并后格式）：
//   1. story_frame：故事框架（世界观底色、主线命题、基调）
//   2. volume_map：卷纲（已有章节=回顾段，续写=章级 prose）
//   3. roles：角色设定（含 Current_State 初始状态）
//   4. book_rules：书籍规则
//   5. pending_hooks：伏笔池（startChapter=0 行为初始钩子）
//
// 注意：不输出 rhythm_principles 和 current_state——
//   节奏原则合并进 volume_map 尾段，
//   角色初始状态合并进 roles.Current_State。
func (a *ArchitectAgent) GenerateFoundationFromImport(
    ctx context.Context,
    book *BookConfig,
    chaptersText string,   // buildImportFoundationSource 的输出
    externalContext string, // 可选外部指令
    reviewFeedback string,  // 可选审查反馈（series 模式重生成时传入）
    importMode ImportMode,
) (*ArchitectOutput, error)
```

**system prompt 关键指令**：

```
你是专业的网络小说架构师。从已有章节中反向推导散文密度的基础设定，同时设计续写路径。

## 续写方向
自然延续已有叙事弧线。推进现有冲突、兑现已埋伏笔、引入有机新变数。

## 输出契约
合并后的 5 段 === SECTION: === 结构：story_frame / volume_map / roles / book_rules / pending_hooks。
不要输出 rhythm_principles 或 current_state——
  节奏原则合并进 volume_map 尾段，
  角色初始状态合并进 roles.当前现状，
  初始钩子写在 pending_hooks startChapter=0 行。

所有 prose 必须从资料包中推导，不得臆造。若资料包声明为压缩包，
把章节目录和正文摘录当作基础设定证据；完整章节会在后续回放阶段逐章进入 truth files。
volume_map 中，已有章节作为"回顾段"（一段散文），续写部分写到章级 prose。
伏笔识别以资料包提供的证据为准，尽量完整。
```

**LLM 参数**：`temperature=0.5`

### 4.2 importMode：continuation vs series

| 模式 | 含义 | 续写方向指令 | 审查流程 |
|------|------|-------------|----------|
| `continuation`（默认） | 接着原有剧情写下去 | 自然延续已有叙事弧线，推进现有冲突、兑现已埋伏笔 | 直接采用，不走 Reviewer |
| `series` | 共享世界观但写新故事 | 必须引入新叙事空间——新冲突、新地点、新时间，5章内引爆，50%以上场景新鲜 | 走 Foundation Reviewer 审查循环 |

**series 模式的续写方向指令**：

```
## 续写方向要求
续写必须引入新叙事空间——新冲突、新地点、新时间。
5章内引爆，50%以上场景新鲜。
```

### 4.3 Foundation Reviewer 审查循环

仅 `importMode=series` 时执行审查循环（continuation 模式直接采用第一版）：

```go
// generateAndReviewFoundation 审查循环逻辑（series 模式）：
func (r *PipelineRunner) generateAndReviewFoundation(params GenerateAndReviewParams) (*ArchitectOutput, error) {
    maxRetries := 2 // 默认重试次数
    foundation, err := params.Generate(ctx, "") // 第一次生成，无审查反馈
    if err != nil {
        return nil, err
    }

    for attempt := 0; attempt < maxRetries; attempt++ {
        review := params.Reviewer.Review(ctx, FoundationReviewInput{
            Foundation:    foundation,
            Mode:          "series",
            Language:      params.Language,
            TargetChapters: params.TargetChapters,
        })

        log.Printf("Foundation review: %d/100 %s", review.TotalScore, passStatus(review.Passed))

        if review.Passed {
            return foundation, nil
        }

        // 未通过，带反馈重新生成
        feedback := buildFoundationReviewFeedback(review, params.Language)
        foundation, err = params.Generate(ctx, feedback)
        if err != nil {
            return nil, err
        }
    }

    // 最终审查（不通过也采用，已达最大重试）
    finalReview := params.Reviewer.Review(ctx, ...)
    log.Printf("Foundation final review: %d/100 ACCEPTED (max retries)", finalReview.TotalScore)
    return foundation, nil
}
```

**审查维度**（由 FoundationReviewerAgent 定义）：
- story_frame 完整性
- volume_map 结构合理性
- roles 角色一致性
- book_rules 规则完备性
- pending_hooks 伏笔覆盖度
- 续写方向可行性（series 模式额外检查）

## 五、Step 3：逐章回放重建 truth files

基础设定生成后，需要逐章回放来重建所有 truth files。这一步复用 ChapterAnalyzerAgent 和 WriterAgent。

**前置操作**（Step 1 完成后执行）：

```go
// resetImportReplayTruthFiles 重置 truth files 到种子态
func (r *PipelineRunner) resetImportReplayTruthFiles(bookDir string, language LengthLanguage) error {
    storyDir := filepath.Join(bookDir, "story")

    // 1. 写入 current_state.md 种子（全字段"未设定"）
    seedState := buildImportReplayStateSeed(language) // 所有字段为 "（未设定）"，当前章节=0
    os.WriteFile(filepath.Join(storyDir, "current_state.md"), seedState, 0644)

    // 2. 写入 pending_hooks.md 种子（空表头）
    seedHooks := buildImportReplayHooksSeed(language) // 只有表头，无数据行
    os.WriteFile(filepath.Join(storyDir, "pending_hooks.md"), seedHooks, 0644)

    // 3. 删除所有扩展 truth files 和数据库
    filesToDelete := []string{
        "chapter_summaries.md",
        "subplot_board.md",
        "emotional_arcs.md",
        "character_matrix.md",
        "volume_summaries.md",
        "particle_ledger.md",
        "memory.db", "memory.db-shm", "memory.db-wal",
    }
    for _, f := range filesToDelete {
        os.Remove(filepath.Join(storyDir, f))
    }
    // 删除 state/ 和 snapshots/ 目录
    os.RemoveAll(filepath.Join(storyDir, "state"))
    os.RemoveAll(filepath.Join(storyDir, "snapshots"))
    return nil
}
```

**逐章回放循环**：

```go
// Step 2: 逐章回放
analyzer := NewChapterAnalyzerAgent(agentCtx)
writer := NewWriterAgent(agentCtx)

totalWords := 0
importedCount := 0

for i := startFrom - 1; i < len(input.Chapters); i++ {
    // 检查取消信号
    if err := ctx.Err(); err != nil {
        return nil, err
    }

    ch := input.Chapters[i]
    chapterNumber := i + 1

    // 1. 构建 governed 输入（复用正常写作的 prepareWriteInput）
    governedInput := r.prepareWriteInput(book, bookDir, chapterNumber)

    // 2. ChapterAnalyzerAgent 分析章节，提取 truth file 更新
    //    注意：content 用已有正文，不调用 LLM 生成
    output, err := analyzer.AnalyzeChapter(ctx, &AnalyzeChapterInput{
        Book:           book,
        BookDir:        bookDir,
        ChapterNumber:  chapterNumber,
        ChapterContent: ch.Content, // 已有正文
        ChapterTitle:   ch.Title,
        ChapterIntent:  governedInput.ChapterIntent,
        ContextPackage: governedInput.ContextPackage,
        RuleStack:      governedInput.RuleStack,
    })
    if err != nil {
        return nil, err
    }

    // 3. 组装持久化输出（content=原文，wordCount=实际字数）
    chapterWordCount := countChapterLength(ch.Content, countingMode)
    persistedOutput := &WriteChapterOutput{
        ParsedWriterOutput: output,         // analyzer 提取的 truth file 更新
        Content:            ch.Content,     // 已有正文
        WordCount:          chapterWordCount,
        PostWriteErrors:    nil,
        PostWriteWarnings:  nil,
    }

    // 4. 保存章节文件 + 核心 truth files（state/ledger/hooks）
    writer.SaveChapter(bookDir, persistedOutput, genreProfile.NumericalSystem, language)

    // 5. 保存扩展 truth files（summaries/subplots/emotional_arcs/character_matrix）
    writer.SaveNewTruthFiles(bookDir, persistedOutput, language)

    // 6. 同步 legacy 结构化状态
    r.syncLegacyStructuredStateFromMarkdown(bookDir, chapterNumber, output)
    r.syncNarrativeMemoryIndex(bookID)

    // 7. 更新章节索引
    existingIndex := r.state.LoadChapterIndex(bookID)
    now := time.Now().UTC().Format(time.RFC3339)
    newEntry := ChapterMeta{
        Number:    chapterNumber,
        Title:     output.Title,
        Status:    "imported", // 标记为导入
        WordCount: chapterWordCount,
        CreatedAt: now,
        UpdatedAt: now,
    }
    // 已存在则替换（续跑场景），否则追加
    updatedIndex := upsertChapterEntry(existingIndex, newEntry)
    r.state.SaveChapterIndex(bookID, updatedIndex)

    // 8. 快照状态（支持回滚+续跑）
    r.state.SnapshotState(bookID, chapterNumber)

    importedCount++
    totalWords += chapterWordCount
}

// 9. 标记书籍为 active
if len(input.Chapters) > 0 {
    r.markBookActiveIfNeeded(bookID)
    r.syncCurrentStateFactHistory(bookID, len(input.Chapters))
}
```

**ChapterAnalyzerAgent 的作用**：

ChapterAnalyzerAgent 读取已有正文，像 Writer 一样构建上下文（加载 10 个 truth files + 题材规则 + 记忆检索），然后让 LLM 分析这章正文并输出：
- 章节标题（可能和导入标题不同，以正文为准）
- 章节摘要
- 状态卡更新（current_state.md）
- 资源账本更新（particle_ledger.md）
- 伏笔池更新（pending_hooks.md）
- 子线状态（subplot_board.md）
- 情感弧线（emotional_arcs.md）
- 角色矩阵（character_matrix.md）

**与正常写作的区别**：

| 对比项 | 正常写作 | 导入回放 |
|--------|----------|----------|
| content 来源 | LLM 生成 | 已有正文 |
| 章节状态 | `draft` / `ready-for-review` | `imported` |
| postWriteErrors | 可能有 | 空 |
| postWriteWarnings | 可能有 | 空 |
| Analyzer 调用 | 仅在 normalize/persistence 时 | 每章都调用 |

## 六、断点续跑逻辑

`resumeFrom` 参数支持从任意章节断点续跑，避免回放中途失败导致全部重来。

### 6.1 续跑决策

```go
startFrom := 1
if input.ResumeFrom != nil && *input.ResumeFrom > 1 {
    startFrom = *input.ResumeFrom
}

// Step 1 仅在 startFrom==1 时执行
if startFrom == 1 {
    // 生成基础设定 + 重置 truth files
    r.generateFoundationAndReset(ctx, input, book, bookDir, language)
}

// Step 2 从 startFrom 开始回放
for i := startFrom - 1; i < len(input.Chapters); i++ {
    // 回放每一章...
}
```

### 6.2 续跑前提条件

- **startFrom > 1 时**：跳过 Step 1，直接从 Step 2 开始。此时 truth files 已由之前的运行部分建立，续跑继续往后面章节追加。
- **章节索引去重**：回放时如果章节索引中已存在同号条目（说明上次已部分处理），直接替换而非追加：

```go
func upsertChapterEntry(index []ChapterMeta, entry ChapterMeta) []ChapterMeta {
    for i, e := range index {
        if e.Number == entry.Number {
            index[i] = entry
            return index
        }
    }
    return append(index, entry)
}
```

### 6.3 快照支持续跑

每章回放后调用 `SnapshotState(bookID, chapterNumber)`，将当前 truth files 状态保存到 `story/snapshots/N/` 目录。续跑时：
- Step 1 被跳过（基础设定已存在）
- Step 2 从 `startFrom` 章开始
- 之前的章节已有快照，truth files 保持上次回放后的状态

### 6.4 续跑触发场景

| 场景 | 原因 | 续跑方式 |
|------|------|----------|
| LLM 调用超时 | 网络波动 | resumeFrom=失败章号 |
| 进程被杀 | OOM / 手动终止 | resumeFrom=快照最大章号+1 |
| 章节解析失败 | LLM 输出异常 | resumeFrom=失败章号 |
