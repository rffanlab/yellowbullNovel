## 一、流程概述

上下文组装是写作流水线的第三阶段（Phase 3.5），在章节规划之后、写手写作之前。ComposerAgent 收集所有相关上下文，按预算压缩，产出三样东西：

1. **ContextPackage**：选定的上下文条目列表（source + reason + excerpt），供 writer/continuity/reviser 注入 prompt
2. **RuleStack**：治理规则栈（4 层优先级 + 活动覆盖边），控制写手行为边界
3. **ChapterTrace**：章节追踪（选了哪些源、token 预算、压缩记录），供审计

```
BookConfig + bookDir + chapterNumber + plan(PlanChapterOutput) + contextBudget
                              │
                              ▼
               ┌──────────────────────────┐
               │ collectSelectedContext    │ 收集 12 类上下文源
               │ (12 类源并行读取+检索)    │ → []ContextSource
               └────────────┬─────────────┘
                            │
                            ▼
               ┌──────────────────────────┐
               │ ContextPackageSchema      │ 构建初始 ContextPackage
               └────────────┬─────────────┘
                            │
                            ▼
               ┌──────────────────────────┐
               │ applyContextBudgetIfNeeded│ 预算控制
               │ (token估算 +              │ → 压缩后 ContextPackage
               │  protected/compressible)  │   + compression 记录
               └────────────┬─────────────┘
                            │
              ┌─────────────┼─────────────┐
              ▼             ▼             ▼
    buildGovernedRuleStack  │    buildGovernedTrace
    (4层规则栈+覆盖边)      │    (选定源+token预算+压缩)
              │             │             │
              ▼             ▼             ▼
        writeGovernedRuntimeArtifacts 落盘
        runtime/chapter-0001.context.md
        runtime/chapter-0001.rulestack.json
        runtime/chapter-0001.trace.json
                            │
                            ▼
        ComposeChapterOutput { contextPackage, ruleStack, trace, paths }
```

**核心设计原则**：
1. **12 类上下文源**：覆盖从章节 memo 到记忆检索的全链路，确保 writer 拿到完整且不冗余的上下文
2. **protected 不可压缩**：author_intent / current_focus / 硬状态 / 活跃伏笔证据**绝不压缩**，即使超预算也直接报错
3. **compressible 可语义编译**：章节摘要 / 大纲选段 / 次要记忆可由 LLM 语义编译压缩
4. **语义选段**：story_frame / volume_map 不整篇塞入，由 LLM 按当前章节目标选择相关段落
5. **治理规则栈**：4 层优先级（硬事实 > 作者意图 > 规划 > 当前任务），明确覆盖边

## 二、入口函数签名

```go
// ComposeChapter 是上下文组装流程的入口。
//
// 参数：
//   - book: 书籍配置
//   - bookDir: 书籍根目录
//   - chapterNumber: 当前章节号
//   - plan: PlannerAgent 的产出
//   - contextBudget: 上下文预算（可为 nil，nil 时从 LLM client 推导）
//
// 返回：
//   - *ComposeChapterOutput: 含 contextPackage + ruleStack + trace + 落盘路径
//   - error
func (a *composerAgent) ComposeChapter(
    ctx context.Context,
    input *ComposeChapterInput,
) (*ComposeChapterOutput, error)

type ComposeChapterInput struct {
    Book          *BookConfig
    BookDir       string
    ChapterNumber int
    Plan          *PlanChapterOutput
    ContextBudget *ContextBudget // 可选
}

type ComposeChapterOutput struct {
    ContextPackage *ContextPackage
    RuleStack      *RuleStack
    Trace          *ChapterTrace
    ContextPath    string // runtime/chapter-0001.context.md
    RuleStackPath  string // runtime/chapter-0001.rulestack.json
    TracePath      string // runtime/chapter-0001.trace.json
}
```

## 三、ComposerAgent 设计

### 3.1 输入输出

#### 输入

| 字段 | 类型 | 说明 |
|------|------|------|
| book | `*BookConfig` | 书籍配置（主要用 Language） |
| bookDir | `string` | 书籍根目录 |
| chapterNumber | `int` | 当前章节号 |
| plan | `*PlanChapterOutput` | PlannerAgent 产出（intent + memo） |
| contextBudget | `*ContextBudget` | 上下文预算（可选） |

#### ContextBudget

```go
type ContextBudget struct {
    ContextWindowTokens  int // LLM 上下文窗口大小
    ReservedOutputTokens int // 预留输出 token 数
}
```

`availableInputTokens = ContextWindowTokens - max(0, ReservedOutputTokens)`

若 `contextBudget` 为 nil，则从 LLM client 的 `contextWindow` 和 `maxTokens` 推导（`contextBudgetFromClient`）。

#### 输出

| 字段 | 类型 | 说明 |
|------|------|------|
| contextPackage | `*ContextPackage` | 选定的上下文条目列表 |
| ruleStack | `*RuleStack` | 治理规则栈 |
| trace | `*ChapterTrace` | 章节追踪（审计用） |
| contextPath | `string` | 上下文落盘路径 |
| ruleStackPath | `string` | 规则栈落盘路径 |
| tracePath | `string` | 追踪落盘路径 |

### 3.2 collectSelectedContext：12 类上下文源

`collectSelectedContext` 并行收集 12 类上下文源，返回 `[]ContextSource`。每类源产生 0-N 个条目。

#### 源 1：chapter_memo（章节 memo）

```go
// 来源：plan.memo
// source: "runtime/chapter_memo"
// reason: "Carry the planner's chapter memo into governed writing."
// excerpt: "goal={memo.goal} | golden-opening=true | {memo.body}"
```

这是 planner 产出的 7 段 memo，是 writer 的核心指令。**protected**。

#### 源 2：current_focus（当前聚焦）

```go
// 来源：story/current_focus.md
// source: "story/current_focus.md"
// reason: "Current task focus for this chapter."
// excerpt: 文件全文
```

当前章节的聚焦指令（含局部覆盖/避免/风格）。**protected**。

#### 源 3：author_intent（作者意图）

```go
// 来源：story/author_intent.md
// source: "story/author_intent.md"
// reason: "User's long-term authorial intent and direction — binding, overrides model defaults."
// excerpt: 文件全文
```

用户长期创作意图，绑定级，覆盖模型默认。**protected**。

#### 源 4：audit_drift（审计漂移）

```go
// 来源：story/audit_drift.md
// source: "story/audit_drift.md"
// reason: "Carry forward audit drift guidance from the previous chapter without polluting hard state facts."
// excerpt: 文件全文
```

上一章审计漂移指导。**protected**。

#### 源 5：current_state（当前硬状态）

```go
// 来源：story/current_state.md
// source: "story/current_state.md"
// reason: "Preserve hard state facts referenced by the active chapter brief or hard constraints."
// excerpt: 文件全文
```

硬状态事实（consolidator 每章追加）。**protected**。

#### 源 6：story_frame（语义选段）

```go
// 来源：story/outline/story_frame.md（回退 legacy story_bible.md）
// source: "story/outline/story_frame.md#{heading-anchor}"
// reason: "Preserve canon constraints referenced by the active chapter brief or hard constraints."
// excerpt: 选定段落的 raw markdown
```

不整篇塞入。先按 Markdown 标题切分成 sections，再按语义相关性筛选，最后可选 LLM 语义选段。**protected**（所有 story_frame 段落）。

#### 源 7：volume_map（语义选段）

```go
// 来源：story/outline/volume_map.md（回退 legacy volume_outline.md）
// source: "story/outline/volume_map.md#{heading-anchor}"
// reason: "Anchor the default planning node for this chapter."
// excerpt: 选定段落的 raw markdown
```

同 story_frame 的语义选段逻辑。**protected**。

#### 源 8：parent_canon / fanfic_canon（正典约束）

```go
// 来源：story/parent_canon.md, story/fanfic_canon.md
// source: "story/parent_canon.md" 或 "story/fanfic_canon.md"
// reason: "Preserve parent canon constraints for governed continuation or fanfic writing."
// excerpt: 文件全文
```

同人/续写模式的正典约束。**protected**。两文件各自独立条目，可能都为空。

#### 源 9：近 5 章摘要 trail

```go
// 来源：story/chapter_summaries.md + chapters/ 目录
// 产生 3 个子条目：
//   a) "story/chapter_summaries.md#recent_titles"
//      reason: "Keep recent title history visible to avoid repetitive chapter naming."
//      excerpt: "12: 标题 | 11: 标题 | 10: 标题 | 9: 标题 | 8: 标题"
//   b) "story/chapter_summaries.md#recent_mood_type_trail"
//      reason: "Keep recent mood and chapter-type cadence visible before writing the next chapter."
//      excerpt: "12: 紧张/冲突 | 11: 压抑/过渡 | ..."
//   c) "story/chapters#recent_endings"
//      reason: "Show how recent chapters ended so the writer avoids structural repetition."
//      excerpt: "ch10: 最后一句 | ch11: 最后一句 | ch12: 最后一句"
```

**compressible**。取最近 5 章摘要（chapterNumber 之前），按章号降序。结尾 trail 取最近 3 章的最后一句（≥2 句才产出）。

#### 源 10：伏笔债务（hook debt）

```go
// 来源：plan.memo.threadRefs × memorySelection.activeHooks × chapter_summaries.md
// source: "runtime/hook_debt#{hookId}"
// reason: "含原始种子文本的叙事债务简报。"
// excerpt: "H007（类型，备忘引用旧债，已开5章）| 读者承诺：xxx | 种于第5章：xxx | 推进于第8章：xxx"
```

仅对 `plan.memo.threadRefs` 中引用的 hook 产出。每条含：hook 元信息 + 种子章节摘要 + 最新推进章节摘要。**protected**。

#### 源 11：记忆检索（facts / summaries / hooks / volume_summaries）

通过 `retrieveMemorySelection` 检索，产生 4 类条目：

```go
// a) facts (current_state 事实)
// source: "story/current_state.md#{predicate-anchor}"
// reason: "Relevant current-state fact retrieved for the current chapter goal."
// excerpt: "{predicate} | {object}"

// b) summaries (章节摘要)
// source: "story/chapter_summaries.md#{chapter}"
// reason: "Relevant episodic memory retrieved for the current chapter goal."
// excerpt: "{title} | {events} | {stateChanges} | {hookActivity}"

// c) hooks (伏笔)
// source: "story/pending_hooks.md#{hookId}"
// reason: "Carry forward unresolved hooks that match the chapter focus."
// excerpt: "{type} | {status} | {expectedPayoff} | {payoffTiming} | {notes}"

// d) volume_summaries (卷级摘要)
// source: "story/volume_summaries.md#{anchor}"
// reason: "Carry forward long-span arc memory compressed from earlier volumes."
// excerpt: "{heading} | {content}"
```

检索输入：`goal` + `outlineNode` + `mustKeep`（从 plan.intent 推导的 retrievalHints）。**compressible**（facts 除外，hooks 和 hook_debt 是 protected）。

#### 源 12：角色矩阵

角色矩阵不作为独立上下文源注入 ContextPackage。它在 planner 阶段已被读取并用于生成 memo。composer 阶段不重复读取。

> **注意**：inkos 原始实现中，角色矩阵（character_matrix）在 planner 的 `planChapterMemo` 中读取并注入 memo prompt，但 composer 的 `collectSelectedContext` 不直接收集它。角色信息通过 `chapter_memo`（源 1）间接传递给 writer。

#### 汇总顺序

```
return [
    ...chapterMemoEntry,      // 1
    ...currentFocus,          // 2
    ...authorIntent,          // 3
    ...auditDrift,            // 4
    ...currentState,          // 5
    ...storyFrameEntries,     // 6
    ...volumeMapEntries,      // 7
    ...parentCanon,           // 8a
    ...fanficCanon,           // 8b
    ...trailEntries,          // 9
    ...hookDebtEntries,       // 10
    ...factEntries,           // 11a
    ...summaryEntries,        // 11b
    ...volumeSummaryEntries,  // 11c
    ...hookEntries,           // 11d
]
```

### 3.3 applyContextBudgetIfNeeded：上下文预算控制

若未设 contextBudget 或 contextWindowTokens ≤ 0，跳过预算控制直接返回。

#### 算法步骤

```
1. availableInputTokens = contextWindowTokens - max(0, reservedOutputTokens)

2. totalTokens = estimateSelectedContextTokens(allEntries)
   // 每条 entry 的 token = estimateTextTokens(source + "\n" + reason + "\n" + excerpt)

3. if totalTokens <= availableInputTokens:
       return  // 无需压缩

4. 拆分：
   protectedEntries = filter(isProtectedContextSource, allEntries)
   compressibleEntries = filter(!isProtectedContextSource, allEntries)

5. protectedTokens = estimateSelectedContextTokens(protectedEntries)
   if protectedTokens > availableInputTokens:
       // 致命错误：protected 上下文本身就超预算
       callback({phase: "error", message: "Protected context exceeds available input budget."})
       return error("Protected context exceeds available input budget. 
                     InkOS will not compress protected author intent, current focus, 
                     hard state, or active hook evidence.")

6. if compressibleEntries.length == 0:
       return  // 无可压缩项，超就超了（记录 note）

7. if compiler == nil:
       // 致命错误：超预算但没提供编译器
       callback({phase: "error", message: "no compiler provided"})
       return error("Context exceeds budget but no compiler was provided.")

8. compileBudget = max(1, availableInputTokens - protectedTokens)

9. callback({phase: "start", protectedTokens, compressibleTokens, budgetTokens: compileBudget})

10. compiled = compiler({
        chapterNumber,
        goal,
        language,
        maxInputTokens: compileBudget,
        protectedEntries,    // 作为参照，不编译
        compressibleEntries, // 只编译这部分
    })

11. if compiled == "":
        callback({phase: "error", message: "empty output"})
        return error("Compressible context compiler returned empty output.")

12. callback({phase: "end", ...})

13. 构建压缩后 ContextPackage:
    selectedContext = [
        ...protectedEntries,           // 原样保留
        {
            source: "runtime/compiled-compressible-context",
            reason: "Semantic compilation of lower-priority context after protected context exceeded the input budget.",
            excerpt: compiled,
        },
    ]

14. 返回 compression 记录:
    {
        compiledSource: "runtime/compiled-compressible-context",
        protectedSources: protectedEntries.map(e => e.source),
        compressedSources: compressibleEntries.map(e => e.source),
        protectedTokens,
        compressibleTokens,
        budgetTokens: compileBudget,
    }
```

#### compileCompressibleContext（LLM 语义编译）

```go
// system prompt (zh):
// 你是 InkOS 的语义上下文编译器。
// 只能编译【可压缩上下文】。【受保护上下文】是绑定参照，不得改写、不得替代总结、不得削弱。
// 输出简洁 Markdown，保留来源指针。保留会影响下一章的人名、未兑现承诺、证据、时间点和约束，
// 丢弃低相关噪声。

// user prompt:
// 章节：第{N}章
// 目标：{goal}
// 压缩后目标预算：不超过 {maxInputTokens} 估算输入 tokens
//
// ## 受保护上下文（只作为参照，不要编译它）
// {protectedBlock 或 "（无）"}
//
// ## 可压缩上下文（只编译这一部分）
// {compressibleBlock 或 "（无）"}

// LLM 调用：temperature=0.2, maxTokens=min(8192, max(512, maxInputTokens))
```

### 3.4 buildGovernedRuleStack：治理规则栈

构建 4 层优先级规则栈 + 活动覆盖边。对应 inkos `utils/context-assembly.ts:buildGovernedRuleStack`。

#### 4 层规则

| 层 ID | 名称 | 优先级 (precedence) | 作用域 (scope) | 说明 |
|-------|------|---------------------|---------------|------|
| L1 | hard_facts | 100 | global | 硬事实（不可覆盖） |
| L2 | author_intent | 80 | book | 作者意图（全书级） |
| L3 | planning | 60 | arc | 规划层（arc 级） |
| L4 | current_task | 70 | local | 当前任务（章节级） |

#### sections（Phase 5 源名）

```go
sections: {
    hard:       ["story_frame", "current_state", "book_rules", "roles"],
    soft:       ["author_intent", "current_focus", "volume_map"],
    diagnostic: ["anti_ai_checks", "continuity_audit", "style_regression_checks"],
}
```

#### overrideEdges（允许的覆盖边）

```go
overrideEdges: [
    {from: "L4", to: "L3", allowed: true,  scope: "current_chapter"},  // 当前任务可覆盖规划
    {from: "L4", to: "L2", allowed: false, scope: "current_chapter"},  // 当前任务不可覆盖作者意图
    {from: "L4", to: "L1", allowed: false, scope: "current_chapter"},  // 当前任务不可覆盖硬事实
]
```

#### activeOverrides（动态生成）

从 `plan.intent` 生成活动覆盖：

```go
// mustAvoid → L4 覆盖 L3
for _, item := range plan.Intent.MustAvoid {
    activeOverrides = append(activeOverrides, ActiveOverride{
        From:   "L4",
        To:     "L3",
        Target: fmt.Sprintf("chapter:%d/mustAvoid", chapterNumber),
        Reason: truncateForOverrideReason(item), // ≤80 字符
    })
}

// styleEmphasis → L4 覆盖 L3
for _, item := range plan.Intent.StyleEmphasis {
    activeOverrides = append(activeOverrides, ActiveOverride{
        From:   "L4",
        To:     "L3",
        Target: fmt.Sprintf("chapter:%d/styleEmphasis", chapterNumber),
        Reason: truncateForOverrideReason(item),
    })
}
```

`truncateForOverrideReason`：空白折叠为单空格，超 80 字符截断为 79 字符 + `…`。

### 3.5 buildGovernedTrace：章节追踪

构建 ChapterTrace 供审计。对应 inkos `utils/context-assembly.ts:buildGovernedTrace`。

```go
type ChapterTrace struct {
    Chapter        int
    PlannerInputs  []string       // plan.plannerInputs
    ComposerInputs []string       // [plan.runtimePath]
    SelectedSources []string      // contextPackage.selectedContext.map(e => e.source)
    UsedSkills     []string       // 技能注册表产出（默认空）
    PromptPacks    []string       // 同上
    ContextNeeds   []string       // 同上
    ContextTiers   struct {
        ProtectedSources   []string
        CompressibleSources []string
    }
    TokenBudget    struct {
        ProtectedTokens      int
        CompressibleTokens   int
        TotalSelectedTokens  int
    }
    Compression    *CompressionRecord // 可选，仅压缩时填充
    Notes          []string           // 如 ["compiled-compressible-context"]
}

type CompressionRecord struct {
    CompiledSource    string   // "runtime/compiled-compressible-context"
    ProtectedSources  []string
    CompressedSources []string
    ProtectedTokens   int
    CompressibleTokens int
    BudgetTokens      int
}
```

**构建逻辑**：
- `protectedEntries` = contextPackage 中 `isProtectedContextSource(source)` 为 true 的条目
- `compressibleEntries` = 其余条目
- `protectedTokens` / `compressibleTokens` = 各自 `sumContextTokens`
- `sumContextTokens` = 每条 `estimateTextTokens(source + "\n" + reason + "\n" + excerpt)` 求和

## 四、语义选段算法（LLM 选择相关大纲段落）

story_frame 和 volume_map 不整篇塞入上下文，而是按段落切分后语义筛选。对应 inkos `composer.ts:selectOutlineSectionEntries`。

### 4.1 切分 Markdown 段落

`splitMarkdownSections(content)` 按 `#` 标题切分：

```go
type MarkdownSection struct {
    Heading string // 标题文本（不含 #）
    Raw     string // 含标题行的完整段落
}
```

切分逻辑：
- 按行扫描，遇到 `^(#{1,6})\s+(.+?)\s*$` 开启新 section
- 上一 section 收尾（要求至少有一行非空内容）
- section 的 raw 包含标题行到下一同级或更高级标题之前的内容

### 4.2 确定性筛选（fallback）

先做确定性筛选，确定有候选段落。筛选信号来自 `deriveOutlineSelectionHints(plan)`：

```go
hints := []string{
    plan.Intent.Goal,
    plan.Intent.OutlineNode,
    plan.Intent.ArcContext,
    plan.Intent.MustKeep...,
    plan.Intent.MustAvoid...,
    plan.Intent.StyleEmphasis...,
    plan.Memo.Goal,
    plan.Memo.Body,
    plan.Memo.ThreadRefs...,
}
```

**story_frame 筛选**（`isRelevantStoryFrameSection`）：
- 标题含硬信号词（世界观/底色/铁律/规则/核心冲突/终局/world/tonal/rule/core conflict/endgame）→ 命中
- 否则按 `matchesOutlineHints` 匹配

**volume_map 筛选**（`isRelevantVolumeMapSection`）：
- 标题含当前章号（chapter N / 第N章 / ch N）→ 命中
- 否则按 `matchesOutlineHints` 匹配

**matchesOutlineHints**：
- 对每个 hint 提取匹配词（`extractMatchTerms`：英文 ≥3 字符的 `[a-z0-9]+`，中文 ≥2 字符的 `[\u4e00-\u9fff]+`）
- section 文本命中词数 ≥ min(2, 词数) → true

**fallback**：确定性筛选无命中时：
- volume_map：找标题含当前章号的段落，无则取第一段
- story_frame：取第一段

若提供了 `outlineSectionSelector`（默认由 `ComposerAgent.selectOutlineSections` 实现），对候选段落做 LLM 精选：

```
// 你是 InkOS 的语义大纲选段器。
// 只选择当前章节真正需要的大纲段落。按语义相关性判断，不要按关键词重合机械选择。
// 只返回严格 JSON：{"selectedSources":["..."]}。必须使用候选里的精确 source id；

// 文件：{fileName}

// 目标：{goal}
// 大纲节点：{outlineNode}
//
// 候选段落：
// #1 {source1}
// heading: {heading1}
// {excerpt1}
//
// #2 {source2}
// ...

// LLM 调用：temperature=0.1, maxTokens=1024
```

**解析**（`parseSelectedSources`）：
- 去掉可能的 ```` ```json ... ``` ```` 围栏
- JSON.parse，提取 `selectedSources` 数组
- 失败则尝试提取第一个 `{` 到最后一个 `}` 之间的子串再 parse
- 仍失败返回空数组（回退确定性筛选结果）
- 过滤：只保留候选 source 集合中存在的 id

**slugifyAnchor**：标题转 anchor（小写 + 非字母数字中文转 `-` + 去首尾 `-`），空则用 `"section"`。

**重要**：LLM 语义选段是**质量增强，非硬依赖**。LLM 调用失败或返回畸形 JSON 时，静默回退到确定性筛选结果，不阻断章节生产。

## 五、上下文预算控制算法

### 5.1 token 估算

```go
// estimateTextTokens 估算文本的 token 数。
// 简单实现：len(text) / 3（中英文混合近似）
// 精确实现：调用 LLM provider 的 tokenizer
func estimateTextTokens(text string) int

// estimateSelectedContextTokens 估算一批上下文条目的总 token 数。
func estimateSelectedContextTokens(entries []ContextSource) int {
    total := 0
    for _, entry := range entries {
        parts := []string{entry.Source, entry.Reason}
        if entry.Excerpt != "" {
            parts = append(parts, entry.Excerpt)
        }
        total += estimateTextTokens(strings.Join(parts, "\n"))
    }
    return total
}
```

### 5.2 protected / compressible 拆分

```go
func isProtectedContextSource(source string) bool {
    // 见第七节完整清单
}
```

**protected（绝不压缩）**：
- `runtime/chapter_memo`
- `story/current_focus.md`
- `story/author_intent.md`
- `story/audit_drift.md`
- `story/outline/story_frame.md` 及其 `#anchor` 子段
- `story/story_bible.md`（legacy）
- `story/outline/volume_map.md` 及其 `#anchor` 子段
- `story/volume_outline.md`（legacy）
- `story/parent_canon.md`
- `story/fanfic_canon.md`
- `story/current_state.md` 及其 `#anchor` 子段
- `story/pending_hooks.md#` 子段
- `runtime/hook_debt#` 子段

**compressible（可 LLM 语义编译）**：
- `story/chapter_summaries.md#` 子段（近 5 章摘要 trail）
- `story/volume_summaries.md#` 子段
- `runtime/compiled-compressible-context`（已编译的不再压缩）

### 5.3 预算控制决策树

```
totalTokens <= availableInputTokens?
├── 是 → 不压缩，原样返回
└── 否 → 拆分 protected / compressible
      │
      protectedTokens > availableInputTokens?
      ├── 是 → 致命错误（protected 不可压缩），抛错
      └── 否 → compressibleEntries 为空?
            ├── 是 → 超预算但无可压缩项，记录 note，原样返回
            └── 否 → compiler 为 nil?
                  ├── 是 → 致命错误（超预算但无编译器），抛错
                  └── 否 → 调 LLM 编译 compressible 为单一编译条目
                        │
                        compiled == ""?
                        ├── 是 → 致命错误（编译器返回空），抛错
                        └── 否 → 构建压缩后 ContextPackage
                              selectedContext = [...protected, {compiled-entry}]
                              记录 compression 信息到 ChapterTrace
```

### 5.4 编译后 ContextPackage 结构

```go
&ContextPackage{
    Chapter: original.Chapter,
    SelectedContext: append(
        protectedEntries,  // 原样保留
        ContextSource{
            Source:  "runtime/compiled-compressible-context",
            Reason:  "Semantic compilation of lower-priority context after protected context exceeded the input budget.",
            Excerpt: compiled, // LLM 编译产出
        },
    ),
}
```

### 5.5 回调通知

`onContextCompression` 回调在压缩过程的关键节点触发：

| phase | 触发时机 | 附带数据 |
|-------|---------|---------|
| `start` | 编译开始 | protectedTokens, compressibleTokens, budgetTokens, sources |
| `end` | 编译成功完成 | 同上 |
| `error` | 任何致命错误 | message + 同上 |

回调用于 UI 展示压缩进度和审计记录。
