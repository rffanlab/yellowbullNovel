## 一、流程概述

章节审查循环是写作流水线的第五阶段（Phase 5），在章节写作（Phase 4）之后、持久化之前。它对 WriterAgent 产出的章节正文做质量审计 + 自动修复，确保章节达到发布标准。

```
WriteChapterOutput (来自 Phase 4)
  │  content / wordCount / postWriteErrors
  ▼
┌─────────────────────────────────────────────────┐
│ 步骤1: normalizeIfHardDrift                      │
│  字数超出硬区间时触发 LengthNormalizer 归一化      │
│  (仅在硬偏离时触发，软偏离不触发)                  │
└─────────────────┬───────────────────────────────┘
                  ▼
┌─────────────────────────────────────────────────┐
│ 步骤2: assess() 首轮审计                         │
│  ┌─────────────────────────────────────────┐    │
│  │ ContinuityAuditor.auditChapter          │    │
│  │   33+ 维度 LLM 审计 (temperature=0.3)   │    │
│  │   → AuditResult {passed, issues, score} │    │
│  └─────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────┐    │
│  │ analyzeAITells (正则检测 AI 痕迹)        │    │
│  │ → []AuditIssue                           │    │
│  └─────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────┐    │
│  │ analyzeSensitiveWords (block/warn 两级)  │    │
│  │ → []AuditIssue                           │    │
│  └─────────────────────────────────────────┘    │
│  ┌─────────────────────────────────────────┐    │
│  │ runPostWriteChecks (确定性检查+伏笔账本)  │    │
│  │ → []AuditIssue                           │    │
│  └─────────────────────────────────────────┘    │
│  合并所有 issues → auditResult                   │
│  lengthInRange 硬门（字数是否在硬区间内）         │
└─────────────────┬───────────────────────────────┘
                  ▼
┌─────────────────────────────────────────────────┐
│ 步骤3: isPassed 判定                             │
│  passed && score >= 85 && lengthInRange          │
│  → true: 直接通过，跳过修复                       │
│  → false: 进入循环修复                           │
└─────────────────┬───────────────────────────────┘
                  ▼ (未通过时)
┌─────────────────────────────────────────────────┐
│ 步骤4: 循环修复 (默认1轮，可配置)                 │
│  for iteration in 0..maxReviewIterations:        │
│    ReviserAgent.reviseChapter(mode="auto")       │
│    → revisedContent                              │
│    assess(revisedContent, temp=0) 重新审计       │
│    if isPassed: 退出循环                         │
│    if 净提升>=3: 继续                            │
│    else: 退出循环                                │
└─────────────────┬───────────────────────────────┘
                  ▼
┌─────────────────────────────────────────────────┐
│ 步骤5: 选择最佳版本                              │
│  snapshots.reduce:                               │
│    1. lengthInRange 优先于 out-of-range          │
│    2. score 更高者优先（需 >= best+3）            │
│  若最佳版本 != 当前版本 → 回退到最佳版本          │
└─────────────────┬───────────────────────────────┘
                  ▼
          ChapterReviewCycleResult
```

**核心设计原则**：
1. **字数独立处理**：字数问题不混入 reviser 的 issues，由 normalizeIfHardDrift 专步处理
2. **多源审计合并**：LLM 审计 + AI 痕迹 + 敏感词 + 确定性检查四源合并
3. **lengthInRange 硬门**：字数不在硬区间内时，即使 score≥85 也不通过
4. **净提升阈值**：修复后分数提升 <3 分则退出循环，避免无效重试
5. **最佳版本回退**：如果修复反而降分，回退到历史最高分版本

## 三、审查循环步骤

### 3.1 步骤1：字数归一化 normalizeIfHardDrift

仅在字数**硬偏离**（超出 hardMin-hardMax 区间）时触发，软偏离不触发。

```go
// normalizeIfHardDrift 检查字数是否硬偏离，如果是则触发归一化
func normalizeIfHardDrift(content string, lengthSpec *LengthSpec) (*normalizeResult, error) {
    wordCount := countChapterLength(content, lengthSpec.CountingMode)
    if !isOutsideHardRange(wordCount, lengthSpec) {
        // 在硬区间内，不触发归一化
        return &normalizeResult{
            Content:   content,
            WordCount: wordCount,
            Applied:   false,
        }, nil
    }
    // 硬偏离：触发 LengthNormalizer 归一化
    result := normalizeDraftLengthIfNeeded(content)
    totalUsage = addUsage(totalUsage, result.TokenUsage)
    return result, nil
}
```

**关键点**：
- `isOutsideHardRange(wordCount, lengthSpec)` 判断字数是否超出 `[hardMin, hardMax]`
- 归一化由独立的 `LengthNormalizer` Agent 处理，不混入 reviser 的 issues
- 归一化后重新计算字数，并标记 `normalizeApplied = true`

### 3.2 步骤2：首轮审计 assess

`assess` 是审查循环的核心函数，合并四个审计源的 issues：

```go
// assess 对章节内容做全面审计
func (c *reviewCycle) assess(ctx context.Context, content string, options *assessOptions) (*assessment, error) {
    // 2a. ContinuityAuditor LLM 审计（33+维度）
    llmAudit := c.auditor.AuditChapter(ctx, &AuditChapterInput{
        BookDir:       c.bookDir,
        Content:       content,
        ChapterNumber: c.chapterNumber,
        Genre:         c.book.Genre,
        Options: mergeControlInput(c.reducedControlInput, options),
    })
    totalUsage = addUsage(totalUsage, llmAudit.TokenUsage)

    // 2b. AI 痕迹检测（正则，零 LLM 成本）
    aiTellsResult := c.analyzeAITells(content)

    // 2c. 敏感词检测（block/warn 两级）
    sensitiveResult := c.analyzeSensitiveWords(content)
    hasBlockedWords := hasBlockSeverity(sensitiveResult.Found)

    // 2d. 确定性检查（段落形状/伏笔账本校验等）
    var postWriteIssues []AuditIssue
    if c.runPostWriteChecks != nil {
        postWriteIssues = c.runPostWriteChecks(content)
    } else {
        // 回退：用 initialOutput.postWriteErrors 转换为 AuditIssue
        postWriteIssues = convertPostWriteErrors(c.initialPostWriteErrors)
    }

    // 2e. 字数硬门判定
    wordCount := countChapterLength(content, c.lengthSpec.CountingMode)
    lengthInRange := !isOutsideHardRange(wordCount, c.lengthSpec)

    // 2f. 合并所有 issues
    allIssues := concatIssues(
        llmAudit.Issues,
        aiTellsResult.Issues,
        sensitiveResult.Issues,
        postWriteIssues,
    )

    // 2g. 判定 passed
    hasPostWriteCritical := hasCriticalSeverity(postWriteIssues)
    passed := (hasBlockedWords || hasPostWriteCritical) ? false : llmAudit.Passed

    // 注意：字数问题不加入 issues，只通过 lengthInRange 硬门控制
    return &assessment{
        AuditResult: &AuditResult{
            Passed:       passed,
            Issues:       allIssues,
            Summary:      llmAudit.Summary,
            ParseFailed:  llmAudit.ParseFailed,
            OverallScore: llmAudit.OverallScore,
        },
        Score:         llmAudit.OverallScore, // 0-100
        LengthInRange: lengthInRange,
    }, nil
}
```

**四个审计源**：

| 审计源 | 类型 | 说明 |
|--------|------|------|
| ContinuityAuditor | LLM | 33+维度结构审稿，输出 passed/issues/overallScore |
| analyzeAITells | 正则 | AI 写作痕迹检测（句式规律/套话/转折词/明喻过载） |
| analyzeSensitiveWords | 正则 | 敏感词检测，block 级直接判 false |
| runPostWriteChecks | 正则 | 确定性检查（段落形状/章节号/禁止句式/伏笔账本） |

**passed 判定逻辑**：
- 如果有 block 级敏感词 → `passed = false`
- 如果有 critical 级 postWrite 问题 → `passed = false`
- 否则 → `passed = llmAudit.Passed`（LLM 自己判定的 passed）

### 3.3 步骤3：通过判定 isPassed

```go
const (
    PassScoreThreshold      = 85 // 通过分数阈值
    NetImprovementEpsilon   = 3  // 净提升阈值
    DefaultMaxReviewIterations = 1 // 默认最大修复轮次
)

// isPassed 判定审计是否通过
func isPassed(a *assessment) bool {
    return a.AuditResult.Passed && a.Score >= PassScoreThreshold && a.LengthInRange
}
```

**三个条件必须同时满足**：
1. `passed == true`（无 critical 问题）
2. `overallScore >= 85`
3. `lengthInRange == true`（字数在硬区间内）

**parseFailed 特殊处理**：如果首轮审计 `parseFailed == true`（LLM 输出无法解析为 JSON），直接跳过自动修复，返回原始内容——避免基于不可靠的审计结果误改正文。

### 3.4 步骤4：循环修复

```go
// 步骤4: 循环修复（默认1轮，可配置）
func (c *reviewCycle) repairLoop(ctx context.Context, initial *assessment, snapshots []reviewSnapshot) (*assessment, []reviewSnapshot, error) {
    currentAudit := initial
    postReviseCount := 0

    if !isPassed(initial) {
        maxIter := c.maxReviewIterations // 默认1
        for iteration := 0; iteration < maxIter; iteration++ {
            // 4a. ReviserAgent 修复（mode="auto"）
            reviser := c.createReviser()
            reviseOutput := reviser.ReviseChapter(ctx, &ReviseChapterInput{
                BookDir:       c.bookDir,
                Content:       c.finalContent,
                ChapterNumber: c.chapterNumber,
                Issues:        currentAudit.AuditResult.Issues,
                Mode:          ReviseModeAuto,
                Genre:         c.book.Genre,
                Options:       mergeControlInput(c.reducedControlInput, c.lengthSpec),
            })
            totalUsage = addUsage(totalUsage, reviseOutput.TokenUsage)

            // 4b. 修复未产出新内容 → 退出
            if reviseOutput.RevisedContent == "" || reviseOutput.RevisedContent == c.finalContent {
                break
            }

            // 4c. 表面归一化 + 重新审计
            revisedContent := c.normalizePostWriteSurface(reviseOutput.RevisedContent)
            revisedWordCount := countChapterLength(revisedContent, c.lengthSpec.CountingMode)
            nextAssessment := c.assess(ctx, revisedContent, &assessOptions{Temperature: 0})

            // 4d. 记录快照
            snapshots = append(snapshots, reviewSnapshot{
                Content:        revisedContent,
                WordCount:      revisedWordCount,
                AuditResult:    nextAssessment.AuditResult,
                Score:          nextAssessment.Score,
                LengthInRange:  nextAssessment.LengthInRange,
            })

            // 4e. 通过 → 退出
            if isPassed(nextAssessment) {
                c.finalContent = revisedContent
                c.finalWordCount = revisedWordCount
                currentAudit = nextAssessment
                break
            }

            // 4f. 净提升 >= 3 → 继续下一轮
            if nextAssessment.Score >= currentAudit.Score + NetImprovementEpsilon {
                c.finalContent = revisedContent
                c.finalWordCount = revisedWordCount
                currentAudit = nextAssessment
                // continue to next iteration
            } else {
                // 4g. 无净提升 → 退出
                break
            }
        }
    }
    return currentAudit, snapshots, nil
}
```

**循环退出条件**：
1. `isPassed == true` → 通过，退出
2. 净提升 < 3 分 → 无改善，退出
3. 修复未产出新内容（空或与原文相同）→ 退出
4. 达到 `maxReviewIterations` 上限 → 退出

### 3.5 步骤5：选择最佳版本

```go
// 步骤5: 从所有快照中选择最佳版本
func selectBestSnapshot(snapshots []reviewSnapshot, currentAudit *assessment) (*reviewSnapshot, bool) {
    best := snapshots[0]
    for _, snap := range snapshots[1:] {
        // 1. lengthInRange 优先
        if snap.LengthInRange != best.LengthInRange {
            if snap.LengthInRange {
                best = snap
            }
            continue
        }
        // 2. score 更高者优先（需 >= best + 3）
        if snap.Score >= best.Score + NetImprovementEpsilon {
            best = snap
        }
    }

    // 如果最佳版本 != 当前版本，且回退条件满足 → 回退
    shouldRestore := best.Content != currentAudit.Content && (
        (best.LengthInRange && !currentAudit.LengthInRange) ||
        best.Score >= currentAudit.Score + NetImprovementEpsilon,
    )
    return best, shouldRestore
}
```

**选择规则**：
1. **lengthInRange 优先**：在区间内的版本优先于不在区间内的版本
2. **分数优先**：同区间内，分数更高的优先（需超出至少3分）
3. **回退条件**：最佳版本与当前版本不同，且满足（在区间而当前不在区间）或（分数高出≥3）

**回退场景**：修复后分数反而下降了，但之前的某个版本分数更高 → 回退到那个更高分版本。

## 四、AuditResult 和 AuditIssue 结构体

```go
// AuditResult 审计结果
type AuditResult struct {
    Passed       bool            // 是否通过（无 critical 问题）
    Issues       []AuditIssue    // 所有问题列表
    Summary      string          // 一句话审计结论
    ParseFailed  bool            // 审计输出是否解析失败（true 时不可自动修复）
    OverallScore *int            // 0-100 综合质量分（可为 nil）
    TokenUsage   *TokenUsage     // token 用量
}

// AuditIssue 审计问题
type AuditIssue struct {
    Severity     string          // "critical" | "warning" | "info"
    Category     string          // 维度名称（如"OOC检查"、"时间线检查"）
    Description  string          // 具体问题描述
    Suggestion   string          // 修改建议
    RepairScope  string          // "local" | "structural" | "unknown"（路由提示）
}
```

**RepairScope 路由提示**（用于 ReviserAgent 的 auto 模式分流）：

| RepairScope | 含义 | Reviser 处理方式 |
|-------------|------|-----------------|
| `local` | 措辞/段落形状/小重复/句段级小修 | 走 PATCHES（局部补丁） |
| `structural` | 主线偏离/时间线断裂/场面缺失/人设崩/视角失败 | 走 REVISED_CONTENT（全章重写） |
| `unknown` | 无法判断 | 让 reviser 自行选择 |

**severity 含义**：

| Severity | 含义 | 影响 |
|----------|------|------|
| `critical` | 严重问题（人设崩/主线偏/伏笔未收等） | `passed = false`，必须修复 |
| `warning` | 警告级别（疲劳词/段落过碎等） | 不影响 passed，但会计入修复 |
| `info` | 参考建议（文笔问题等） | 不影响 passed，不驱动修复路由 |

| 36 | 关系动态 | Relationship Dynamics Check | 同人：关系发展是否合理 |
| 37 | 正典事件一致性 | Canon Event Consistency Check | 同人：是否与原作事件时间线矛盾 |

**维度 #6 伏笔检查的 Phase 7 升级规则**：

- `critical` 级别仅适用于 `promoted=true` 的伏笔
- `promoted=true` 且 `core_hook=true` 的伏笔过期超过 10 章 → warning 升级为 critical
- `promoted=true` 的受阻伏笔，"已阻 Y 章"且 Y≥6 → warning
- 卷尾仍有 `promoted=true` 主线伏笔未回收且无"延至下一卷"规划 → critical
- 非 promoted 的 stale 伏笔 → info 级，不判失败

**parseAuditResult 多策略容错**：

```go
// parseAuditResult 解析审计输出（4种策略容错）
func parseAuditResult(content string, language Language) *AuditResult {
    // 策略1: 提取平衡的 JSON 对象（不贪婪）
    if balanced := extractBalancedJson(content); balanced != "" {
        if result := tryParseAuditJson(balanced, language); result != nil {
            return result
        }
    }

    // 策略2: 整个内容作为 JSON 解析
    if strings.HasPrefix(strings.TrimSpace(content), "{") {
        if result := tryParseAuditJson(content, language); result != nil {
            return result
        }
    }

    // 策略3: 提取 ```json 代码块
    if match := regexCodeBlock.FindStringSubmatch(content); len(match) > 1 {
        if result := tryParseAuditJson(match[1], language); result != nil {
            return result
        }
    }

    // 策略4: 正则提取单个字段（最后回退）
    if passedMatch := regexPassed.FindStringSubmatch(content); len(passedMatch) > 1 {
        return parseAuditFieldsFallback(content, passedMatch[1], language)
    }

    // 全部失败 → parseFailed
    return &AuditResult{
        Passed:      false,
        ParseFailed: true,
        Issues:      []AuditIssue{{Severity: "critical", Category: "系统错误", ...}},
        Summary:     "审稿输出解析失败",
    }
}
```

**评分校准**（system prompt 中的指导）：

| 分数区间 | 含义 |
|----------|------|
| 95-100 | 可直接发布，无明显问题 |
| 85-94 | 有小瑕疵但整体流畅可读，读者不会出戏 |
| 75-84 | 有明显问题但故事主干完整，需要修但不紧急 |
| 65-74 | 多处影响阅读体验的问题，节奏或连续性有断裂 |
| <65 | 结构性问题，需要大幅重写 |

**审稿边界（硬约束）**：
- 审稿只审**完成度 + 结构**，不审文笔
- 文笔问题只能标 `severity="info"` 供 Polisher 参考
- 不计入 `passed` / `overallScore`，绝不标为 `critical`
- 稀疏 memo 合法（喘息章/后效章），不判 incomplete

func RunChapterReviewCycle(ctx context.Context, params *ChapterReviewCycleParams) (*ChapterReviewCycleResult, error) {
    cycle := &reviewCycle{
        params:     params,
        totalUsage: params.InitialUsage,
        finalContent: params.InitialOutput.Content,
        finalWordCount: params.InitialOutput.WordCount,
    }

    // 步骤1: 字数归一化
    normalized := cycle.normalizeIfHardDrift(params.InitialOutput.Content)
    cycle.finalContent = normalized.Content
    cycle.finalWordCount = normalized.WordCount
    cycle.normalizeApplied = normalized.Applied

    // 步骤2: 首轮审计
    initial := cycle.assess(ctx, cycle.finalContent, nil)
    snapshots := []ReviewSnapshot{{
        Content:       cycle.finalContent,
        WordCount:     cycle.finalWordCount,
        AuditResult:   initial.AuditResult,
        Score:         initial.Score,
        LengthInRange: initial.LengthInRange,
    }}

    // parseFailed 特殊处理
    if initial.AuditResult.ParseFailed {
        return cycle.buildResult(initial, snapshots, false), nil
    }

    // 步骤3+4: 循环修复
    currentAudit, snapshots := cycle.repairLoop(ctx, initial, snapshots)

    // 步骤5: 选择最佳版本
    best, shouldRestore := selectBestSnapshot(snapshots, currentAudit)
    if shouldRestore {
        cycle.finalContent = best.Content
        cycle.finalWordCount = best.WordCount
        currentAudit = &assessment{
            AuditResult:   best.AuditResult,
            Score:         best.Score,
            LengthInRange: best.LengthInRange,
        }
    }

    revised := len(snapshots) > 1 && cycle.finalContent != params.InitialOutput.Content
    return cycle.buildResult(currentAudit, snapshots, revised), nil
}
