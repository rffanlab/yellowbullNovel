## 一、流程概述

章节规划是写作流水线的第二阶段（Phase 3），在建书骨架之后、上下文组装之前。PlannerAgent 为下一章产出两样东西：

1. **ChapterIntent**（确定性）：goal + outlineNode + arcContext + mustKeep/mustAvoid/styleEmphasis。由宿主从大纲/状态/作者意图确定性推导，不用 LLM。
2. **ChapterMemo**（LLM 生成）：7 段 Markdown 结构，指导 writer 写正文。LLM 生成 + 严格解析器，最多重试 3 次，失败生成 fallback memo。

```
BookConfig + bookDir + chapterNumber + externalContext
                    │
                    ▼
          ┌─────────────────────┐
          │  步骤1: 加载种子材料  │ loadPlanningSeedMaterials
          │  (大纲/状态/作者意图/  │ → authorIntent, currentFocus, storyBible,
          │   前章结尾/摘要)      │   volumeOutline, currentState, brief,
          └──────────┬──────────┘   previousEndingExcerpt, recentSummaries
                     │
                     ▼
          ┌─────────────────────┐
          │  步骤2: 定位大纲节点  │ findOutlineNode(volumeOutline, chapterNumber)
          │  (精确/范围/锚点匹配) │ → outlineNode string
          └──────────┬──────────┘
                     │
                     ▼
          ┌─────────────────────┐
          │  步骤3: 推导章节目标  │ deriveGoal (优先级链)
          │  (外部→局部→大纲→    │ → goal string
          │   聚焦→作者→默认)    │
          └──────────┬──────────┘
                     │
                     ▼
          ┌─────────────────────┐
          │  步骤4: 收集约束      │ collectMustKeep/MustAvoid/StyleEmphasis
          │  (状态+圣经 / 聚焦+  │ → mustKeep[], mustAvoid[], styleEmphasis[]
          │   禁忌 / 作者意图)   │
          └──────────┬──────────┘
                     │
                     ▼
          ┌─────────────────────┐
          │  步骤5: 聚合材料+检索 │ gatherPlanningMaterials
          │  (记忆检索 facts/    │ → memorySelection{hooks,summaries,facts,
          │   summaries/hooks)  │   volumeSummaries, recyclableHooks}
          └──────────┬──────────┘
                     │
                     ▼
          ┌─────────────────────┐
          │  步骤6: 构建Intent    │ ChapterIntentSchema
          │  (确定性, 不调LLM)   │ → ChapterIntent
          └──────────┬──────────┘
                     │
                     ▼
          ┌─────────────────────┐
          │  步骤7: LLM生成Memo  │ planChapterMemo (≤3次重试)
          │  (7段Markdown)       │ → ChapterMemo
          │  失败→fallback memo  │
          └──────────┬──────────┘
                     │
                     ▼
          intent.goal = memo.goal  (用 LLM 的具体目标覆盖推导的 fallback)
                     │
                     ▼
          renderIntentMarkdown → 落盘 runtime/chapter-0001.intent.md
                     │
                     ▼
          PlanChapterOutput { intent, memo, intentMarkdown, plannerInputs, runtimePath }
```

**重试策略**：memo 解析失败最多重试 3 次。每次失败把错误信息追加到 user 消息末尾让 LLM 自我修正。3 次都失败则生成 degraded fallback memo（含 warning），不崩溃整条章节流水线。

## 二、入口函数签名

```go
// PlanChapter 是章节规划流程的入口。
//
// 参数：
//   - book: 书籍配置
//   - bookDir: 书籍根目录
//   - chapterNumber: 要规划的章节号（从 1 开始）
//   - externalContext: 外部创作指令（可为空）
//
// 返回：
//   - *PlanChapterOutput: 包含 intent + memo + intentMarkdown
//   - error
func (a *plannerAgent) PlanChapter(
    ctx context.Context,
    input *PlanChapterInput,
) (*PlanChapterOutput, error)

type PlanChapterInput struct {
    Book           *BookConfig
    BookDir        string
    ChapterNumber  int
    ExternalContext string // 可选
}

type PlanChapterOutput struct {
    Intent        *ChapterIntent  // 确定性推导的章节意图
    Memo          *ChapterMemo    // LLM 生成的 7 段 memo
    IntentMarkdown string         // 渲染后的 intent.md 全文
    PlannerInputs []string        // planner 读取的输入文件路径列表
    RuntimePath   string          // 落盘路径 runtime/chapter-0001.intent.md
}
```

## 三、PlannerAgent 7 步流程

### 步骤 1：loadPlanningSeedMaterials

读取规划所需的种子材料。对应 inkos `utils/planning-materials.ts:loadPlanningSeedMaterials`。

**读取的文件**（并行读取，不存在返回 `"(文件尚未创建)"` 占位）：

| 文件路径 | 字段 | 说明 |
|---------|------|------|
| `story/author_intent.md` | authorIntent | 用户长期创作意图 |
| `story/current_focus.md` | currentFocus | 当前章节聚焦（含局部覆盖/避免/风格） |
| `story/outline/story_frame.md` | storyBible | 世界观散文骨架（Phase 5，回退 legacy `story_bible.md`） |
| `story/outline/volume_map.md` | volumeOutline | 分卷散文地图（Phase 5，回退 legacy `volume_outline.md`） |
| `story/chapter_summaries.md` | chapterSummariesRaw | 章节摘要表 |
| `story/book_rules.md` | bookRulesRaw | 规则卡 |
| `story/current_state.md` | currentState | 硬状态（Phase 5 回退：从 roles + pending_hooks 种子行推导） |
| `story/brief.md` | brief | 用户创作 brief（最高优先级） |
| `chapters/XXXX.md` (前一章) | previousEndingExcerpt | 前一章最后 320 字 |

**输出结构**：

```go
type PlanningSeedMaterials struct {
    StoryDir              string
    AuthorIntent          string
    CurrentFocus          string
    StoryBible            string // story_frame 内容
    VolumeOutline         string // volume_map 内容
    BookRulesRaw          string
    CurrentState          string
    ChapterSummariesRaw   string
    Brief                 string
    OutlineNode           string // 步骤2 填充
    RecentSummaries       []StoredSummary // 最近 4 章，按章号升序
    PreviousEndingHook    string // 前一章的 hookActivity
    PreviousEndingExcerpt string // 前一章最后 320 字
}
```

**前一章结尾摘录逻辑**：
- `previousChapter = chapterNumber - 1`，若 < 1 则无
- 读取 `chapters/` 目录下以 `{previousChapter:04d}` 开头的 `.md` 文件
- 去掉第一行（标题），取正文最后 320 字

### 步骤 2：findOutlineNode

在 `volumeOutline`（volume_map.md 全文）中定位当前章对应的卷级节点。对应 inkos `planner.ts:findOutlineNode`。

**三级匹配策略**（按优先级）：

#### 2.1 精确匹配（Exact）

在每一行查找精确章号标记：

```
正则1：/^(?:#+\s*)?(?:[-*]\s+)?(?:\*\*)?Chapter\s*{N}(?!\d|\s*[-~–—]\s*\d)(?:[:：-])?(?:\*\*)?\s*(.*)$/i
正则2：/^(?:#+\s*)?(?:[-*]\s+)?(?:\*\*)?第\s*{N}\s*章(?!\d|\s*[-~–—]\s*\d)(?:[:：-])?(?:\*\*)?\s*(.*)$/i
```

- `{N}` 是当前章号
- `(?!\d|\s*[-~–—]\s*\d)` 排除范围标记（如 "Chapter 10-20"）
- 匹配到后：先取行内内容（冒号后），无内容则取下一行非标题非空内容

#### 2.2 范围匹配（Range）

精确匹配失败，查找章号范围：

```
正则：/^(?:#+\s*)?(?:[-*]\s+)?(?:\*\*)?Chapter\s*(\d+)\s*[-~–—]\s*(\d+)\b.../i
正则：/^(?:#+\s*)?(?:[-*]\s+)?(?:\*\*)?第\s*(\d+)\s*[-~–—]\s*(\d+)\s*章.../i
```

- 检查 `chapterNumber` 是否在 `[min(start,end), max(start,end)]` 范围内
- 匹配到后：取行内内容；无内容则提取该范围所属的 section（向上找最近的 `#` 标题，向下到同级标题），并在 section 内按 `beatIndex = chapterNumber - rangeStart` 提取编号 beat（`1. xxx` / `2. xxx`）

#### 2.3 锚点匹配（Anchor）

精确和范围都失败，找到第一个锚点行（任意精确或范围标记），取其行内内容或下一行内容，然后 break。

#### 2.4 兜底

返回 `volumeOutline` 的第一个非标题非空非模板占位行（`extractFirstDirective`）。

### 步骤 3：deriveGoal

目标推导遵循**优先级链**（从高到低，第一个命中即返回）：

| 优先级 | 来源 | 提取方式 | 说明 |
|--------|------|---------|------|
| 1 | 外部指令 (externalContext) | `extractFirstDirective` | 取第一个非 `#`/非 `-`/非模板占位行 |
| 2 | 局部覆盖 (currentFocus 的 local override 段) | `extractLocalOverrideGoal` | 在 current_focus.md 里找 `## local override` / `## 局部覆盖` / `## 本章覆盖` 等标题段，取其列表项或首条指令 |
| 3 | 大纲节点 (outlineNode) | `extractFirstDirective` | 步骤2 的大纲节点文本 |
| 4 | 当前聚焦 (currentFocus 的 focus 段) | `extractFocusGoal` | 在 current_focus.md 里找 `## active focus` / `## 当前聚焦` 等段，取列表项（`;`/`；`连接）或首条指令 |
| 5 | 作者意图 (authorIntent) | `extractFirstDirective` | 取首条指令 |
| 6 | 默认 | 硬编码 | `"Advance chapter {N} with clear narrative focus."` |

**extractFirstDirective**：按行扫描，跳过 `#` 开头、`-` 开头、模板占位（如 `(describe ...)` / `（在这里描述...）`），返回第一个有内容的行。

**extractLocalOverrideGoal**：先 `extractSection` 提取指定标题段，再从段内提取列表项（`-` 开头，最多 3 条，用 `；` 或 `; ` 连接），无列表则取首条指令。

**extractSection**：按 Markdown 标题层级解析。匹配目标标题（normalizeHeading 后比较），收集该标题下到同级或更高级标题之前的内容。

### 步骤 4：collectMustKeep / MustAvoid / StyleEmphasis

#### collectMustKeep(currentState, storyBible)

```go
func collectMustKeep(currentState, storyBible string) []string {
    items := append(
        extractListItems(currentState, 2),  // 从 currentState 提取 "-" 列表项，最多2条
        extractListItems(storyBible, 2)..., // 从 storyBible 提取 "-" 列表项，最多2条
    )
    return unique(items)[:4] // 去重后最多4条
}
```

#### collectMustAvoid(currentFocus, prohibitions)

```go
func collectMustAvoid(currentFocus string, prohibitions []string) []string {
    // 1. 从 current_focus 找 avoid 段
    avoidSection := extractSection(currentFocus, []string{"avoid", "must avoid", "禁止", "避免", "避雷"})
    var focusAvoids []string
    if avoidSection != "" {
        focusAvoids = extractListItems(avoidSection, 10)
    } else {
        // 无 avoid 段，扫描全文中含 avoid/don't/不要/别/禁止 的 "-" 行
        focusAvoids = scanAvoidLines(currentFocus)
    }
    // 2. 合并 prohibitions（book_rules 的禁止事项）
    return unique(append(focusAvoids, prohibitions...))[:6]
}
```

#### collectStyleEmphasis(authorIntent, currentFocus)

```go
func collectStyleEmphasis(authorIntent, currentFocus string) []string {
    return unique(append(
        extractFocusStyleItems(currentFocus, 3), // current_focus 的 focus 段列表项
        extractListItems(authorIntent, 2)...,    // authorIntent 的列表项
    ))[:4]
}
```

**prohibitions 来源**：通过 `readAuthoritativeBookRules(bookDir)` 读取结构化规则（Phase 5 优先 `outline/story_frame.md` frontmatter，回退 `book_rules.md`），取 `rules.prohibitions`。

### 步骤 5：gatherPlanningMaterials + 记忆检索

对应 inkos `utils/planning-materials.ts:gatherPlanningMaterials`。

```go
func gatherPlanningMaterials(params) *PlanningMaterials {
    seed := params.seed // 步骤1 的产出
    if seed == nil {
        seed = loadPlanningSeedMaterials(params.bookDir, params.chapterNumber)
    }

    // 记忆检索：根据 goal/outlineNode/mustKeep 检索相关记忆
    memorySelection := retrieveMemorySelection(RetrieveMemorySelectionInput{
        BookDir:       params.bookDir,
        ChapterNumber: params.chapterNumber,
        Goal:          params.goal,
        OutlineNode:   params.outlineNode,
        MustKeep:      params.mustKeep,
    })

    return &PlanningMaterials{
        PlanningSeedMaterials: seed,
        OutlineNode:           params.outlineNode,
        ActiveHooks:           memorySelection.ActiveHooks,
        MemorySelection:       memorySelection,
        PlannerInputs: []string{
            "story/author_intent.md",
            "story/current_focus.md",
            "story/outline/story_frame.md",
            "story/outline/volume_map.md",
            "story/chapter_summaries.md",
            "story/book_rules.md",
            "story/current_state.md",
            "story/pending_hooks.md",
            // + memorySelection.dbPath（如有）
        },
    }
}
```

**MemorySelection 包含**：
- `activeHooks`：活跃伏笔列表（status != resolved && status != deferred）
- `recyclableHooks`：陈旧伏笔（必须本章 advance/resolve/defer）
- `summaries`：相关章节摘要
- `facts`：相关 current_state 事实
- `hooks`：相关 pending_hooks 条目
- `volumeSummaries`：相关卷级摘要

**plannerInputs** 是 planner 读取过的文件路径列表，写入 ChapterTrace 供审计。

### 步骤 6：ChapterIntentSchema 构建（确定性 Intent）

```go
arcContext := buildArcContext(book.Language, seedMaterials.VolumeOutline, outlineNode)
// arcContext = "卷纲节点：" + outlineNode（若 outlineNode 为空或 volumeOutline 是占位则 nil）

intent := &ChapterIntent{
    Chapter:        chapterNumber,
    Goal:           goal,                  // 步骤3 推导的 fallback goal
    OutlineNode:    outlineNode,           // 步骤2 的大纲节点
    ArcContext:     arcContext,            // 卷纲上下文
    MustKeep:       mustKeep,              // 步骤4
    MustAvoid:      mustAvoid,             // 步骤4
    StyleEmphasis:  styleEmphasis,         // 步骤4
}
// ChapterIntentSchema 校验：chapter >= 1, goal 非空
```

**关键**：Intent 的 goal 此时是推导的 fallback。步骤 7 的 LLM memo 产出后，`intent.goal = memo.goal` 会用 LLM 的具体目标（≤50 字）覆盖它。

### 步骤 7：planChapterMemo - LLM 生成 7 段 memo

#### 7.1 准备 LLM 输入

并行读取 5 个文件：

| 文件 | 读取函数 | 用途 |
|------|---------|------|
| `story/roles/` 或 `character_matrix.md` | readCharacterMatrix | 角色矩阵 |
| `story/subplot_board.md` | readSubplotBoard | 支线看板 |
| `story/emotional_arcs.md` | readEmotionalArcs | 情感弧线 |
| `story/pending_hooks.md` | readPendingHooks | 伏笔表 |
| `story/book_rules.md` | readBookRules (结构化) | 规则卡 |

**readBookRules（结构化）**：通过 `readStructuredBookRules` 读 YAML frontmatter，渲染为紧凑 Markdown：
```
- 主角 林辞 / 人设锁：冷静、缜密、记仇 / 行为约束：不杀平民、不背叛盟约
- 本书禁忌：
  - 禁止机械降神
  - 禁止反派降智
- 题材锁：xianxia / 禁止混入：科幻、克苏鲁
```

#### 7.2 构建 user 消息

用模板 `PLANNER_MEMO_USER_TEMPLATE` 填充 12 个占位符：

| 占位符 | 来源 | 说明 |
|--------|------|------|
| `{{chapterNumber}}` | 当前章号 | |
| `{{brief_block}}` | brief | 用户创作 brief 段（最高优先级） |
| `{{chapter_context_block}}` | externalContext | 本章用户指令段 |
| `{{previous_chapter_ending_excerpt}}` | previousEndingExcerpt | 前章最后一屏（无则"本章为起始章"） |
| `{{recent_summaries}}` | formatRecentSummaries(前3章) | 最近 3 章摘要表 |
| `{{current_arc_prose}}` | composeCurrentArcProse | 活跃支线+近期情感线 |
| `{{protagonist_matrix_row}}` | extractProtagonistRow | 主角行 |
| `{{opponent_rows}}` | extractOpponentRows(3) | 主要对手行 |
| `{{collaborator_rows}}` | extractCollaboratorRows(3) | 主要协作者行 |
| `{{relevant_threads}}` | extractRelevantThreads | 活跃伏笔+支线 |
| `{{recyclable_hooks}}` | formatRecyclableHooks | 必须回收的陈旧 hook |
| `{{isGoldenOpening}}` | 黄金开篇判定 | 是/否 |
| `{{book_rules_relevant}}` | 结构化规则卡 | |

**黄金三章指引**（chapterNumber ≤ 3 时追加）：
```
## 黄金三章规划指引 — 第 {N} 章

第1章：把主角直接抛进核心冲突
第2章：让金手指通过具体事件落地
第3章：钉下 3-10 章可达成的短期目标
场景 ≤ 3 个、人物 ≤ 3 个、信息分层强制
```

#### 7.3 LLM 调用 + 重试循环

```
maxRetries = 3
currentUserMessage = userMessage

for attempt := 0; attempt < 3; attempt++ {
    response := chat(systemPrompt, currentUserMessage, temperature=0.7)

    memo, err := parseMemo(response.content, chapterNumber, isGoldenOpening)
    if err == nil {
        return memo  // 成功
    }

    // 解析失败，追加错误反馈重试
    lastError = err
    currentUserMessage = userMessage + "\n\n## 上次输出的错误\n" + err.message + "\n请修正后重新输出。"
}

// 3 次都失败，生成 fallback memo
return parseMemo(buildFallbackMemoMarkdown(...), ...)
```

#### 7.4 parseMemo 解析器

**必需的 8 个 section**（zh/en 双标题，匹配任一即可）：

| # | zh 标题 | en 标题 | 最小内容字数 |
|---|--------|---------|-------------|
| 1 | `## 本章目标` | `## Chapter goal` | （goal 提取，非空即可） |
| 2 | `## 关联线索` | `## Thread refs` | （可为"无"） |
| 3 | `## 当前任务` | `## Current task` | 20 |
| 4 | `## 读者此刻在等什么` | `## What the reader...` | 20 |
| 5 | `## 该兑现的 / 暂不掀的` | `## To pay off / to keep buried` | 20 |
| 6 | `## 日常/过渡承担什么任务` | `## What the slow / transitional beats carry` | 20 |
| 7 | `## 关键抉择过三连问` | `## Three-question check...` | 20 |
| 8 | `## 章尾必须发生的改变` | `## Required end-of-chapter change` | 20 |
| 9 | `## 本章 hook 账` | `## Hook ledger for this chapter` | 20 |
| 10 | `## 不要做` | `## Do not` | 1 |

**解析步骤**：
1. `stripWrappingFence`：去掉外层 ```` ```md ... ``` ```` 代码块围栏
2. `dropLeadingProse`：去掉 memo 标题前的废话（"好的，下面是..."）
3. `extractGoal`：从 `## 本章目标` / `## Chapter goal` 段取第一句（按 `\n` 或 `。` 或 `. ` 分割），≤50 字（超长截断为 47 字 + `...`）
4. `extractThreadRefs`：从 `## 关联线索` 段提取形如 `[A-Za-z][A-Za-z0-9_-]*\d+` 的 ID（如 H03、S004），去重
5. `extractMemoBody`：从第一个必需 section 标题开始截取到末尾
6. 检查所有必需 section 标题存在（zh 或 en 任一）
7. 检查每个 section 内容 ≥ minContentChars（`extractSectionContent` 提取标题到下一个 `\n## ` 之间的内容）
8. 失败抛 `PlannerParseError`，含具体缺失/空段信息

#### 7.5 fallback memo

3 次重试都失败时生成 degraded memo（确保流水线不崩溃）：

```markdown
# 第 {N} 章 memo

## 本章目标
{fallbackGoal 或 "按当前大纲继续推进第 N 章"}
```

### 黄金开篇判定

```go
func isGoldenOpeningChapter(language string, chapterNumber int) bool {
    if strings.HasPrefix(strings.ToLower(language), "zh") || language == "" {
        return chapterNumber <= 3  // 中文 ≤ 3 章
    }
    return chapterNumber <= 5      // 英文 ≤ 5 章
}
```

黄金开篇章会在 user 消息末尾追加 `## 黄金三章规划指引` 段，对每章规定硬槽位：
- 第 1 章：主角出场即面对主线矛盾（追杀、死局、被夺权、穿越即危机）
- 第 2 章：金手指通过具体事件落地（不是旁白"他觉醒了"，而是"他用了 XX，发生了 YY"）
- 第 3 章：钉下 3-10 章可达成的短期目标
- 开篇精简：场景 ≤ 3 个、人物 ≤ 3 个、信息分层强制

### 伏笔预算

在 `renderIntentMarkdown` 中渲染伏笔预算段：

```go
func renderHookBudget(activeCount int, language string) string {
    cap := 12
    if activeCount < 10 {
        return fmt.Sprintf("### 伏笔预算\n- 当前 %d 条活跃伏笔（容量：%d）", activeCount, cap)
    }
    remaining := max(0, cap-activeCount)
    return fmt.Sprintf("### 伏笔预算\n- 当前 %d 条活跃伏笔——接近容量上限（%d）。"+
        "仅剩 %d 个新坑位。优先回收旧债，不要轻易开新线。", activeCount, cap, remaining)
}
```

**活跃伏笔计数**：`activeHooks` 中 `status != "resolved" && status != "deferred"` 的数量。

**告警阈值**：活跃伏笔 ≥ 10 条时在 intent.md 里显示接近上限警告。

## 四、ChapterMemo 的 7 段结构详解

ChapterMemo 的 body 包含以下 7+2 段（前 2 段是 goal/threadRefs，后 7 段是必需 memo 段，最后 1 段是禁止事项）：

### 段 1：本章目标 (`## 本章目标`)
- ≤ 50 字
- LLM 产出的具体任务陈述（如"把七号门被动过手脚钉成现场实证"）
- 解析后赋值给 `intent.goal`，覆盖推导的 fallback

### 段 2：关联线索 (`## 关联线索`)
- Markdown 列表，从输入的 pending_hooks/subplot_board 中挑出的 ID
- 没有写"无"
- 解析为 `memo.threadRefs []string`

### 段 3：当前任务 (`## 当前任务`)
- 一句话：本章主角要完成的**具体动作**，不要抽象描述
- 示例："林秋潜入药园，在七号门门栓上找到焦痕并用炭笔拓印"

### 段 4：读者此刻在等什么 (`## 读者此刻在等什么`)
- 两行：
  1. 读者现在期待什么（基于前几章的埋伏）
  2. 本章对这个期待做什么——制造更强缺口 / 部分兑现 / 完全兑现 / 暂不兑现但给暗示

### 段 5：该兑现的 / 暂不掀的 (`## 该兑现的 / 暂不掀的`)
- 列表：
  - 该兑现：X → 兑现到什么程度
  - 暂不掀：Y → 先压住，留到第 N 章

### 段 6：日常/过渡承担什么任务 (`## 日常/过渡承担什么任务`)
- 非高压章节：每段非冲突段落说明功能，格式 `[段落位置] → [承担功能]`
- 高压/冲突章节：写"不适用 - 本章无日常过渡"

### 段 7：关键抉择过三连问 (`## 关键抉择过三连问`)
- 主角本章最关键的一次选择：
  - 为什么这么做？
  - 符合当前利益吗？
  - 符合他的人设吗？
- 对手/配角本章最关键的一次选择：同上三问

### 段 8：章尾必须发生的改变 (`## 章尾必须发生的改变`)
- 1-3 条，从以下维度选：
  - 信息改变 / 关系改变 / 物理改变 / 权力改变

### 段 9：本章 hook 账 (`## 本章 hook 账`)
- 4 个子分类，每个用 `-` 列表：
  - `open:` 新钩子（≤30字，≤2个，推荐揭1埋2）
  - `advance:` 推进已有钩子（状态迁移）
  - `resolve:` 回收钩子
  - `defer:` 显式延后（需写理由）
- **硬规则**：
  - pressured/near_payoff 且 ≥5 章未推进的 hook **必须** advance 或 resolve，不允许 defer
  - hook_id 必须真实存在于输入 pending_hooks
  - 纯高压章至少 1 条 advance 或 defer
  - 当前任务对应某 hook 兑现时必须在 resolve 声明

### 段 10：不要做 (`## 不要做`)
- 2-4 条硬约束

## 六、LLM prompt 要点

### 6.1 system prompt 核心原则（15 条内化规则）

```
你是这本小说的创作总编，职责是为下一章产生一份 chapter_memo。你不写正文——你只规划这章要完成什么、
兑现什么、不要做什么。下游写手（writer）会按你的 memo 扩写正文。

你的工作原则（内化，不要在 memo 里引用条目号）：

1. 3-5 章一个小目标周期：每 3-5 章必须有一个小目标达成或悬念升级
2. 主动塑造读者期待：作者刻意制造"还没兑现但快要兑现"的缺口，兑现时必须超过读者预期 70%
3. 万物皆饵：日常/过渡章节的每一笔都要是未来剧情的伏笔或钩子
4. 人设防崩：角色行为由"过往经历 + 当前利益 + 性格底色"共同驱动。禁止反派突然降智、主角突然圣母
5. 1 主线 + 1 支线：支线必须为主线服务，不同时推 3 条以上支线
6. 爽点密集化：每 3-5 章一个小爽点，全员智商在线
7. 高潮前铺垫：大高潮前 3-5 章必须有线索埋设
8. 高潮后影响：爆发章之后 1-2 章必须写出改变
9. 人物立体化：核心标签 + 反差细节 = 活人
10. 五感具体化：场景描写必须有具体可视化感官细节
11. 钩子承接：每章章尾留钩
12. 钩子账本必须结账：每章对活跃 hook 做明确动作（open/advance/resolve/defer）
13. 圆心法同场多视角：核心事件把多个角色聚到同场景时，每个关键角色安排独立内心反应
14. 揭 1 埋 2 推荐：每 resolve 1 个钩子，尽量 open 2 个新钩子（上限 ≤ 2 个/章），硬底线是 open ≥ resolve
15. 用户设定的内容比例必须落成场面：不要只复述比例，要拆成可见场景
```

### 6.3 hook 账硬规则

```

- 输入的 pending_hooks 里如果有任何 hook 状态已是 "pressured" 或 "near_payoff"
```
